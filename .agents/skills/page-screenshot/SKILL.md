# 页面截图验证 Skill

## 触发条件

当用户提出以下任一意图时触发本 Skill：

- "截个图"
- "截图验证一下"
- "验证一下这个页面"
- "看一下页面效果"
- "浏览器验证"
- "页面渲染怎么样"
- "对 XXX.html 截图"
- "把 XXX 页面截个图"

## 执行步骤

### Step 1：确定目标页面路径

根据用户输入，按优先级确定要截图的 HTML 文件：

1. **用户明确指定完整路径**：直接使用用户提供的相对路径或绝对路径。
2. **用户只提供页面名称或功能名**：
   - 在当前项目目录下搜索 `03-高保真页面/`（或用户提及的目录）
   - 使用 `Glob` 查找匹配文件，例如搜索 `**/*control*.html`
   - 若存在多个匹配，列出候选文件请用户确认
3. **用户未指定任何页面**：
   - 询问用户需要截图的页面路径

### Step 2：执行截图命令

使用 PowerShell 在当前项目目录下执行全局 CLI：

```powershell
page-screenshot --file "<目标文件路径>" --out "07-bugs" --prefix "<功能前缀>"
```

参数说明：
- `--file`：目标 HTML 文件路径（必需）
- `--out`：截图输出目录，默认为当前项目的 `07-bugs/`（与项目规范一致）
- `--prefix`：截图文件名前缀，建议从功能名提取，例如 `road_control`、`filing_form`
- `--width` / `--height`：可选，默认 1366×768
- `--root`：可选，静态服务器根目录，默认当前工作目录

### Step 3：读取并展示结果

1. 命令执行成功后，会输出截图保存路径。
2. 使用 `Read` 工具读取生成的 PNG 图片并展示给用户。
3. 向用户报告：截图保存路径、页面地址、截图分辨率。

### Step 4：失败处理

若命令执行失败：
1. 检查错误信息是否包含"未找到系统 Chrome 或 Edge"
   - 若是，提示用户安装 Chrome/Edge 或使用 `--chrome-path` 指定路径
2. 检查是否包含"文件不存在"
   - 若是，重新确认文件路径
3. 其他错误，展示原始错误信息并建议用户直接运行命令排查

## 路径推断规则

| 用户输入示例 | 推断行为 |
|---|---|
| "control.html" | 搜索 `**/*control*.html` |
| "道路资源动态管控" | 搜索 `**/*control*.html` 或匹配页面标题 |
| "road/catalog/control.html" | 直接使用该相对路径 |
| "03-高保真页面/road/catalog/control.html" | 直接使用该相对路径 |
| "截个图" | 询问用户具体页面 |

## 输出格式

执行完成后按以下格式回复：

```
已使用 page-screenshot 对 {页面路径} 进行浏览器截图验证。

- 截图保存路径：{完整路径}
- 页面访问地址：{http://127.0.0.1:8000/...}
- 分辨率：{宽度}x{高度}

[截图展示]
```

## 注意事项

- 本 Skill 仅调用已全局安装的 `page-screenshot` 命令，不需要 AI 自身具备截图能力。
- 截图过程会临时启动 Chrome/Edge 和本地 HTTP 服务，运行结束后自动清理。
- 若页面引用了上级目录资源（如 `../../common.js`），务必确保 `--root` 设置为项目根目录，否则相对路径资源可能加载失败。
