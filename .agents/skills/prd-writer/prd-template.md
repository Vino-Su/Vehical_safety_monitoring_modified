# PRD撰写结构模板

> 本模板基于车辆管理系统5份实际PRD文档提炼总结，并经4轮优化调整形成。适用于B端企业管理系统的产品需求文档撰写。

---

## 核心原则

1. **表格驱动**：所有结构化信息均使用表格呈现，避免冗长段落
2. **逻辑先行形式后置**：功能详细设计先讲业务逻辑，再讲呈现形式
3. **权限融入操作**：权限规则与按钮功能描述结合，不在功能详细设计中单独成章
4. **枚举值就地定义**：枚举值直接写入表单字段定义的"说明"列，不设独立章节
5. **通用组件抽离复用**：分页、导出、删除、变更日志等通用组件集中定义在附录，正文仅引用
6. **章节弹性扩展**：各章节标注【可根据实际需求补充】，允许按模块特性灵活增补
7. **名词先解释再使用**：关键术语在名词解释章节统一说明，确保全文理解一致
8. **子章节灵活省略**：功能详细设计的3.x.1~3.x.7为通用框架，按实际功能需要取舍，省略后编号连续不跳号
9. **文档格式极致紧凑**：仅允许单个表格后空一行以提升可读性，其他位置禁止插入空行，大章节间仅使用`---`分隔线
10. **以页面为维度划分功能点**：每个独立页面单独编写3.x章节；仅作为按钮操作的功能（删除、启用/停用、导出等）合并到其所在页面的功能章节中，不单独成章

## 🔑 功能详细设计编排规则

**功能点划分原则：**
- **独立页面**：每个独立页面（含弹窗页面）作为一个功能点单独编写3.x章节
- **按钮级功能**：仅作为按钮操作的功能（如删除、启用/停用、导出等）不单独成章，合并到其所在页面的功能章节中
- **合并方式**：在所在页面章节的"交互流程"或"操作列按钮"中描述其完整交互逻辑；若按钮功能较复杂（有独立的前置校验、弹窗规格等），可在所在页面章节内以加粗小标题形式补充说明，如：**删除操作**、**导出操作**
- **Tab页展开规则**：当页面中包含Tab页签时，每个Tab页使用`####`四级标题展开，标题格式为`#### 3.x.y.z {Tab名称}Tab`（z从1开始递增，y为Tab所在的子章节编号），在四级标题下按需编写该Tab的表单字段、交互流程等内容，无论Tab复杂度高低统一采用此方式

**子章节取舍规则（按需包含，省略后编号连续不跳号）：**

| 子章节 | 是否必需 | 包含条件 |

| 3.x.1 页面布局 | 必需 | 每个功能点必须有 |

| 3.x.7 弹窗规格 | 可省略 | 有弹窗时包含 |

**示例1：** 某功能点仅有页面布局+交互流程+弹窗规格，则编号为3.x.1、3.x.2、3.x.3（不跳号）

**示例2：** 字典列表页面包含查询、列表展示、编辑弹窗、删除操作、启用/停用操作、导出操作，则：
- 3.x为"字典列表管理"（而非将删除、启用/停用、导出各拆为独立3.x章节）
- 交互流程中分别描述各按钮操作的完整交互
- 删除操作和启用/停用操作较复杂（有二次确认），以加粗小标题**删除操作**、**启用/停用操作**在交互流程中补充
