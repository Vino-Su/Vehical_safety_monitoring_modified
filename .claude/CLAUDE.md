# 项目规则

## 0. 总体原则
所有生成的内容（页面、PRD、数据）必须与`02-需求文档`中的文件保持一致，并html风格严格遵循`.claude`中的UI_design文件。

## 1. 需求与设计一致性
- 绘制的每个页面都必须与PRD文档中的要求完全对应，不得遗漏功能点。
- 所有页面（包括弹窗、二级页面、状态提示等）必须遵循工作区UI设计规范，包含颜色、字体、图标、间距、圆角等。
- 即便PRD文档未明确要求，但按照业务逻辑明显存在的页面（如：表单提交后的成功弹窗、删除确认框、详情页、编辑页等），也必须绘制。

## 2. PRD撰写与变更
- PRD撰写需要调用prd-writer skill进行撰写
- 用户提出新需求或变更时，同时更新：
  - 对应的HTML页面（按新需求重新绘制）
  - PRD文档（位于 `02-产品文档`）
- **版本记录规则**：同一天的更新合并为一条版本记录
  
## 3. 数据与示例
- 填充示例数据时，优先参照 `01-需求文档\示例数据参考.md` 中的内容（如果文件存在且字段匹配）。
- 若无该文件或无对应字段，可自行生成合理示例数据（符合字段类型，如姓名、时间、金额等）。

## 4. 自测与质量保证
- **重大版本新建**或**问题修复后**，必须打开浏览器进行实际验证（点击、跳转、弹窗、数据展示等）。
- 浏览器截图统一放置在 `07-bugs` 文件夹中，命名建议：`功能名_测试时间.png`。
- **举一反三**：当发现一个问题（如某个列表没有分页、某个弹窗样式错误），需主动检查所有其他页面是否存在同类问题，并一并修复。
- 每次绘制完成后，需对照需求文档和UI设计规范进行一次完整检查，确保无遗漏、无偏差。

## 5. 备份机制
- 以下情况触发备份：(1) 用户明确要求 (2) 修改超过3个文件 (3) 生成新版本页面，备份内容包括：
  - 03-高保真页面
  - 02-产品文档
  - 01-需求文档（如有修改）
  - index.html
- 备份路径：`06-备份`
- 使用时间戳命名文件夹，如 backup_20260608_143022，保留最近3次，自动删除更早的。
- 如需恢复，将`06-备份`中对应时间戳文件夹的内容复制回原目录

## 6. 项目结构与Github Pages适配
项目目录结构必须遵循以下规范（用于上传至Github并启用Pages生成公开链接）：
项目根目录/
├── index.html # 入口文件，必须放在根目录
├── README.md # 项目说明文档，需定期更新
├── 01-需求文档/ # 用户提供的需求文档（仅当用户提及时读取）
├── 02-产品文档/ # 系统自动生成的PRD文档
├── 03-高保真页面/ # 所有绘制的HTML页面及CSS、JS等组件
├── 05-参考文件/ # 用户提供的参考文件（仅当用户提及时读取）
├── 06-备份/ # 最近一次的备份内容
└── 07-bugs/ # 自测截图
> 根目录下的 `index.html` 直接跳转到 `03-高保真页面` 中的layout页面即可（建议使用iframe或导航菜单）。
> 每次生成新版本页面或者有重大变更后，应同步更新 README.md。

## 7. 对话结束输出
每次对话的最后一句话必须以`good job！`结尾
