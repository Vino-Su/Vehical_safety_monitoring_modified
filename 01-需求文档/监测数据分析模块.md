## 1. 模块概述

监测数据分析模块基于测试示范全量数据，提供测试数据管理、查询检索、事故分析研判及三大专题（道路测试、示范应用、示范运营）的可视化分析功能。所有分析图表均支持**时间范围选择、维度筛选、图形导出及数据下钻**。

***

## 2. 功能详细设计

### 2.1 测试数据分析与管理

#### 2.1.1 测试数据总览

**页面结构**：顶部为时间筛选栏，下方为统计卡片行，接着是图表区、当日测试情况、企业车辆统计表和实时车辆动态列表。

**时间筛选栏**：

- 快捷选项：今日、昨日、最近7天、最近30天、本月
- 支持自定义起止日期选择器
- 默认选中"最近7天"
- 累计数据和图表共用同一时间筛选

**累计统计卡片**（单行4个卡片）：

| 卡片标题 | 数据字段 | 数据来源 | 备注 |
| ---- | ---- | ---- | ---- |
| 测试企业数 | enterprise\_count | 企业信息表按状态为"已审批"统计 | 随筛选周期变化 |
| 临牌发放数 | license\_count | 临牌发放记录表 | 随筛选周期变化 |
| 测试时长(小时) | total\_test\_hours | 车辆运行记录表 | 求和，随筛选周期变化 |
| 下发提醒(次) | total\_remind\_count | 提醒记录表 | 随筛选周期变化 |

**图表区**（两列布局）：

1. **测试主体累计测试里程**：横向柱状图，X轴为里程(km)，Y轴为企业名称，显示各企业在所选周期内的累计里程。
2. **测试里程趋势分析**：组合图，主坐标轴为折线图（自动驾驶里程、人工驾驶里程），副坐标轴为自动驾驶占比折线（虚线），时间粒度按筛选范围适配（今日/昨日按时段，最近7天按日，最近30天/本月按周）。

**当日测试情况**（6个紧凑卡片）：

| 卡片标题 | 说明 |
| ---- | ---- |
| 当前测试企业数 | 今日有测试活动的企业数 |
| 测试车辆数 | 今日参与测试车辆总数 |
| 当日测试里程(km) | 今日累计测试里程 |
| 当日测试时长(小时) | 今日累计测试时长 |
| 载人测试总里程(km) | 载人测试累计里程 |
| 载人测试总时长(小时) | 载人测试累计时长 |

- 带手动刷新按钮，点击可模拟实时数据波动

**当前测试企业车辆统计表**：

| 列名 | 字段 | 说明 |
| ---- | ---- | ---- |
| 企业名称 | company\_name | |
| 当日测试车辆数 | test\_vehicles | 该企业当日测试车辆总数 |
| 正在测试车辆数 | active\_vehicles | 自动驾驶/人工驾驶且车速>0的车辆数，绿色标签展示 |

**实时车辆动态列表**：

| 列名 | 字段 | 说明 |
| ---- | ---- | ---- |
| 临时号牌 | plate | 蓝色高亮 |
| 所属企业 | enterprise | |

| 最后上报时间 | time | |

- 默认每页5条，支持分页
- 8秒自动轮换刷新（模拟新上报数据）

#### 2.1.2 异常告警情况统计

> **原始需求**：对车辆运行告警与车载终端异常告警数据进行融合处理与分类汇聚，构建多级告警分级与统计分析体系，对不同类型异常事件的发生频次、影响范围及持续时长进行结构化展示，实现对测试运行过程中设备异常与行为异常的集中监测与风险态势感知。

**页面结构**：3个Tab切换（告警概览、告警趋势、企业排名）

> **V1.2变更说明**：回归原始需求核心诉求，从5Tab调整为3Tab（告警明细合并入告警概览、去掉分析报告Tab、去掉预测模型、去掉东风悦享专用分析）；增加设备异常/行为异常融合分类维度；补全影响范围和持续时长核心指标。

**Tab1 - 告警概览**：

**筛选栏**：

- 开始日期/结束日期选择器（默认最近7天）
- 告警大类筛选：全部/设备异常/行为异常（**新增**，对应融合分类）
- 告警类型筛选：全部/车辆运行异常/临牌有效期预警/电子围栏越界/终端通信异常/设备故障告警/其他告警
- 告警级别筛选：全部/严重/一般/提示
- 所属企业筛选
- 车牌号筛选（输入框，模糊匹配）
- 查询和导出按钮
- 告警大类与告警类型联动：选择"设备异常"时，告警类型自动筛选为终端通信异常、设备故障告警；选择"行为异常"时，告警类型自动筛选为车辆运行异常、临牌有效期预警、电子围栏越界、其他告警

**统计卡片**（单行8个）：

1. 当日告警总数（红色#ff4d4f）
2. 设备异常告警（橙色#fa8c16）— 终端通信异常+设备故障告警数量
3. 行为异常告警（蓝色#1677ff）— 车辆运行异常+临牌有效期预警+电子围栏越界+其他告警数量
4. 已处理告警（绿色#52c41a）
5. 告警处理率
6. 严重告警占比（红色#ff4d4f）
7. 平均持续时长（紫色#722ed1）— **新增**
8. 影响车辆数（青色#13c2c2）— **新增**

**图表区**（3行2列布局）：

1. **告警大类分布**：双环形嵌套饼图，内环展示设备异常/行为异常两大类占比，外环展示各细分类型占比。
2. **告警级别分布**：分组柱状图，按设备异常/行为异常分组展示严重/一般/提示三级数量。
3. **告警趋势（近30天）**：面积折线图，展示告警总数随时间变化趋势，含平均线标记。
4. **影响范围分析**：组合图（柱状+折线），柱状展示每日影响车辆数，折线展示累计影响企业数。
5. **持续时长分布**：箱线图，展示各类告警的持续时长分布（P25/P50/P75）。
6. **告警热点地图**：Leaflet地图，随机生成60个点位，按级别以不同颜色圆形标记展示（严重红色/一般橙色/提示蓝色），支持点击弹窗显示详情。

**告警明细列表**（12列）：

| 列名 | 字段 | 说明 |
| ---- | ---- | ---- |
| 告警编号 | id | ALT-日期-序号格式 |
| 时间 | time | |
| 车牌号 | plate | |
| VIN码 | vin | 17位编码格式 |
| 所属企业 | enterprise | |
| 告警类型 | type | |
| 告警大类 | category | 设备异常（橙色标签）/行为异常（蓝色标签） |
| 告警级别 | level | 严重（红色标签）、一般（黄色标签）、提示（蓝色标签） |
| 地点 | location | 10个预设路段点位 |
| 持续时长 | duration | 如"2h 15min"，未结束显示"进行中" |
| 处理状态 | status | 待处理（蓝色）、处理中（黄色）、已完成（绿色） |
| 操作 | action | "详情"按钮，弹出详情弹窗 |

- 默认每页10条，支持分页（带省略号省略的页码导航）
- 详情弹窗使用Ant Design描述列表布局（两列），展示完整告警信息：
  - 基本信息：告警编号、告警时间、告警类型、告警大类、告警级别、车辆号牌、VIN码、所属企业、告警地点、处理状态
  - 扩展信息：持续时长、影响范围（关联同类型告警数量、波及车辆数）、关联事件、数据来源、告警详情

**Tab2 - 告警趋势**：

- 时间范围筛选（最近7天/30天/90天快捷切换）
- 告警大类筛选（全部/设备异常/行为异常）
- 告警频次趋势：多折线图（设备异常/行为异常/总数三线，含同比虚线）
- 告警级别趋势：堆叠面积图（严重/一般/提示三级堆叠）
- 持续时长趋势：折线图（各类告警平均持续时长变化）
- 告警类型环比：分组柱状图（各类型当期vs上期，标注环比变化率）
- 趋势汇总表（告警类型/当期数量/上期数量/环比变化/平均持续时长/影响车辆数）

**Tab3 - 企业排名**：

- 快捷时间按钮：今日、昨日、最近7天、最近30天、本月
- 对比企业多选（选择2-4家企业进行对比分析）
- 企业告警排名 Top10 横向柱状图（按告警数降序，≥30标红色，≥20标橙色，其余蓝色）
- 企业告警对比表格（排名/企业名称/告警总数/设备异常/行为异常/严重/一般/提示/处理率/平均持续时长）
- 企业对比分析：根据"对比企业"选择，动态展示选中企业的告警趋势对比折线图 + 告警类型分布对比饼图

2. **事故趋势与处理状态**：堆叠柱状图，X轴为时间（按月），Y轴为事故数，按处理状态（已处理/处理中/未处理）堆叠。

**事故列表**：

| 列名   | 字段             |
| ---- | -------------- |
| 事故时间 | accident\_time |
| 车辆号牌 | plate          |
| 事故类型 | accident\_type |

#### 2.1.4 运行状态分析

**维度选择**：可按“按企业”、“按车型”、“按行政区域”切换。

**图表区**：

1. **里程统计**：柱状图（维度为X轴，里程为Y轴），支持切换“总里程”、“自动驾驶里程”。
2. **时长统计**：同上，Y轴为运行时长(h)。
3. **平均速度分布**：直方图，展示各速度区间车辆数量。
4. **数据采集质量**：表格展示各企业的数据完整率、及时率、异常数据占比。

| 列名     | 说明             |
| ------ | -------------- |
| 企业名称   | <br />         |
| 应上报条数  | 理论值            |
| 实际上报条数 | <br />         |
| 完整率    | 实际上报/应上报\*100% |
| 及时率    | 在规定时间窗内上报占比    |
| 异常数据率  | 字段非法或超出阈值占比    |

#### 2.1.5 预警事件分析

**页面**：时间筛选 + 事件类型筛选 + 图表

1. **事件数量趋势**：折线图，按日/周/月展示各严重程度事件数。
2. **事件-事故关联分析**：双轴图，柱状图为事件数，折线为事故率（事故数/事件数）。

#### 2.1.6 测试里程分析

**视图切换**：可按“单车”、“企业”、“区域”切换分析维度。

- **单车维度**：表格展示每辆车的总里程、自动驾驶里程、人工驾驶里程、最近测试日期，支持排序。
- **企业维度**：柱状图对比企业总里程，支持点击企业下钻到车辆。
- **区域维度**：结合地图，以区域填充色显示各行政区里程密度。

**里程增长趋势**：按所选维度展示月度里程增量趋势（折线图）。

#### 2.1.7 产业发展分析

**页面**：包含多个标准化图表，支持一键导出报告。

1. **企业数量增长**：折线图，展示累计接入企业数和当月新增企业数。
2. **测试规模趋势**：企业数、车辆数、测试里程三项指标随时间变化。
3. **场景渗透率**：雷达图或横向柱状图，展示环卫、物流、公交、出租车等场景的车辆数占比。
4. **综合报告导出**：可将以上图表和数据自动汇总为PDF/Word报告，包含封面、目录、图表及解释文字。

### 2.3 事故数据分析

#### 2.3.1 事故数据解析

**流程**：

1. 测试主体/安全员在“事故数据上报”功能中上传事故处置报告（支持PDF、图片格式）。
2. 系统调用第三方OCR服务接口，根据**预设的固定模板**进行字段识别。模板字段包括：
   - 事故时间
   - 事故地点（路段/坐标）
   - 涉事车辆号牌
   - 事故类型（碰撞、刮擦等）
   - 责任认定（全责/主责/同责/次责/无责）
   - 伤亡情况（轻伤/重伤/死亡人数）
   - 天气、路面状况等环境信息
3. OCR结果与车端上传的碰撞时刻前后30秒数据进行关联，自动填充速度、加速度、制动状态、转向角等运行数据。
4. 生成结构化事故报告，存入数据库，并提供预览和人工校正入口（校正后重新保存，记录校正人及校正时间）。

**解析结果详情页展示**：

| 字段分组  | 字段名                        |
| ----- | -------------------------- |
| 基本信息  | 事故编号、时间、地点、事故类型            |
| 车辆信息  | 号牌、VIN、所属企业、自动驾驶等级、当时驾驶模式  |
| 运行数据  | 碰撞前速度、最大减速度、转向状态、制动状态、油门开度 |
| 环境信息  | 天气、光照、路面状况                 |
| 责任与伤亡 | 责任认定、轻伤人数、重伤人数、死亡人数        |
| 附件    | 原报告文件、现场照片                 |

#### 2.3.2 事故成因分析

**片区事故成因分析**：
查询条件
- 区域选择：下拉选择行政区（如襄城区、樊城区）。
- 时间范围：日期范围选择器。
- 事故类型：多选下拉（碰撞、刮擦、追尾等）。
- 车辆类型：多选下拉。
分析结果展示
统计卡片：
- 总事故数、涉及车辆数、主成因类型、高风险片区（事故密度最高）。
成因分布图：
- 饼图：四大主成因占比（行为因素、车辆因素、道路因素、环境因素）。
- 柱状图：Top10 子因素的事故数量与占比。
- 热力地图：事故点位热力图（基于事故空间坐标）。
交互：支持点击饼图扇区下钻至事故清单，点击表格行查看事故明细。

**路段事故成因分析**：
查询条件
- 道路选择：输入框模糊搜索道路名称（基于道路资源信息建档）。
- 时间范围：日期范围选择器。
路段信息卡片
- 展示道路基本属性：名称、等级、车道数、路段长度、RSU 覆盖率、限速值、交叉口密度。
分析结果展示
- 路段事故成因分布饼图（同上）。
- 道路参数与事故关联图：散点图，X轴为参数（如曲率），Y轴为事故数，支持切换参数。
- 交通流量与事故对比：组合图，柱状图为流量（来自路侧设备），折线为事故数，分析流量高峰与事故相关性。
- 事故时段分布热力表：按小时×星期统计。

**单起事故成因分析**：
分四个区域：
1. 事故基本信息卡片
事故编号、时间、地点、类型、涉事车辆（车牌/VIN）、伤亡情况。
2. 事故还原时间轴
以时间线展示关键事件点（含车端与路侧数据）：
T-30s: 车辆自动驾驶模式，速度 55 km/h（车端）
T-20s: 路侧监测到前方 80m 有行人横穿（路侧激光雷达）
T-15s: 路侧 RSU 广播行人预警信息（路侧 RSU）
T-12s: AD 系统检测到行人，开始减速（车端）
T-8s:  速度降至 35 km/h，信号灯当前相位为绿灯（路侧信号机）
T-0s:  碰撞发生，车速 20 km/h，瞬时减速度峰值 9.8 m/s²（车端）
T+2s:  车辆停止，路侧检测行人状态（路侧）
T+5s:  安全员接管（车端模式切换）
3. 各维度贡献度饼图
展示人、车、路、环境四维度贡献百分比，模型自动计算。
4. 成因详情表格
维度	子因素	贡献度	判定依据（含路侧数据）	关键数据（事故值 vs 标准值）
行为因素	制动时机偏晚	35%	AD系统检测到行人→制动指令间隔 2.5s（正常≤1.5s）	反应时间 2.5s / 标准 1.5s
车辆因素	制动性能不足	25%	最大减速度 5.2 m/s²（正常≥6.5 m/s²）	减速度 5.2 vs 6.5 m/s²
道路因素	视距受限	20%	路侧数据显示碰撞点在弯道内侧，视距仅 80m	有效视距 80m / 标准 120m
环境因素	行人违规穿行	20%	路侧记录行人未走斑马线，且跨越护栏	行人轨迹来自路侧雷达
5. 关键数据对比表
列出关键参数的事故值与标准值，辅助责任认定。

#### 2.3.3 事故特征分析

**事故时空特征分析**：

- 时间维度：提供按小时、星期、月份的事故分布热力表或折线图。
- 空间维度：GIS热力图，支持按事故类型过滤。
- 组合筛选：时间范围、事故类型、区域。

**事故关联检索**：

- 搜索入口：在事故列表页提供高级搜索，可按人（安全员姓名）、车（号牌/VIN/企业）、路（道路名称）、时间（范围）组合查询。
- 结果列表展示事故摘要，点击查看全息档案（包含基本信息、运行数据、成因分析结果、关联的其他事故事件）。

**事故多维特征画像**：

- 以标签云或矩阵图展示“事故类型-车辆类型”、“事故类型-测试场景”等的交叉分布。
- 可切换X/Y轴维度。

**碰撞形态分析**：

- 展示碰撞形态分布饼图。
- 交叉表格：碰撞形态 vs 伤亡等级，单元格显示事故数量。

#### 2.3.4 事故分析研判

**多发地点分析**：

- 地图上标记多发点（可配置阈值，如发生3起以上），鼠标悬停显示事故数量、主要类型。
- 下方列表：点位名称（路段/交叉口）、事故数、主要类型、近期趋势。

**风险区域分析**：

- 地图上按风险等级（高、中、低）填充不同颜色区域。
- 列表展示区域名称、风险等级、评分、主要风险因素。

**事故趋势分析**：

- 折线图展示实际事故数和预测事故数（虚线），**预测参数由模型自动计算，不支持用户调整**。
- 表格列出未来短期（如接下来4周）的预测事故数及置信区间。

**车辆主体分析**：

- 柱状图对比不同品牌/车型的事故率（事故数/测试里程）。
- 表格可排序。

**事故分析报告与管控建议**：

- 提供报告生成向导，包含以下选项：
  - **报告类型**：
    - 周报（固定统计上周一至上周日）
    - 月报（固定统计上个月）
    - **自定义**：选择【开始日期】和【结束日期】，系统生成该**一次性特定时间段**的综合分析报告。本期不支持配置定时重复执行的周期报告（如每双周自动生成）。
  - **包含模块**（可多选）：总体情况、多发地点、风险区域、趋势分析、管控建议。
- 生成后提供在线预览，用户确认后可导出为PDF或Word文档。报告自动排版，包含图表和文字描述。

### 2.4 测试示范数据分析

通用功能：每个专题分析页面顶部提供“分析阶段”切换（道路测试、示范应用、示范运营），页面内容相应变化。

#### 2.4.1 道路测试专题分析

**总体监测**：

- 指标卡片行：测试企业数、测试车辆数、开放道路里程、当月测试里程
- 趋势图：月度企业数、车辆数、里程数组合图
- 行政区对比：地图 + 表格

**企业监测**：

- 企业搜索与列表（名称、属地、性质、测试车辆数、里程、时长、告警数）
- 支持点击企业名称进入企业详情，包含该企业车辆排名、里程趋势等
- 可多选企业进行横向对比（雷达图：里程、时长、车辆数、告警率、脱手率）

**车辆运行分析**：

- 车辆列表（号牌、车型、企业、累计里程、本月里程、接管次数、故障次数）
- 点击车辆进入车辆详情：
  - 基本信息（牌号、VIN、尺寸等）
  - 安全员信息（姓名、干预率）
  - 电子围栏越界记录（时间、围栏名称）
  - 里程排名（在企业内和全局的百分位）
  - **测试场景分析**：使用系统预置场景分类（闯红灯预警、车速引导、紧急制动预警、施工路段提醒等），展示该车辆在不同场景下的里程分布和时长分布（堆叠柱状图）。**不支持自定义场景标签。**
  - **人工接管分析**：
    - 接管次数趋势（按月）
    - 平均接管里程排名
    - 脱离地点地图分布
    - 脱离场景分类饼图
    - 脱离原因分类（目标物占用车道、路口博弈等）饼图
    - 关键脱离目标分类统计（施工牌、隔离栏等）
  - **车辆异常分析**：统计该车异常事件（长时间不移动、在线不稳定等），展示异常高发时间段和地点。**异常判定阈值使用系统默认值（如30分钟无移动视为长时间不移动），暂不提供用户配置。**

#### 2.4.2 示范应用专题分析

**总体监测**：卡片（示范企业数、运营车辆数、覆盖区域数、累计服务次数）
**企业监测**：排名维度增加服务次数、运营时长。
**车辆运行分析**：类似道路测试，增加服务次数、平均载客量（如适用）、示范场景分析（无人出租、小巴、公交、配送等）。

#### 2.4.3 示范运营专题分析

**总体监测**与**企业监测**：比示范应用增加营收相关指标（如总运营收入、每公里收入等，如果企业上报）。
**车辆运行分析**：运营服务次数按场景、线路、站点统计。
**运营收费分析**：

- **收费数据由运营企业手动上传**（Excel模板），包含车辆号牌、服务时间、服务类型、应收金额、实收金额等字段。
- 平台自动与服务次数数据进行一致性比对，对比结果展示：
  - 匹配一致记录数
  - 缺失收费记录的服务次数
  - 金额偏差记录
- 页面提供差异清单表格，并支持导出。

## 3. 通用交互规范

- **时间控件**：统一使用日期范围选择器，支持快捷选项。
- **表格默认排序**：时间倒序或数值降序，需明确标注。
- **导出**：列表页均有“导出”按钮，导出当前筛选条件下的全量结果（上限10万条），超限提示缩小范围。导出格式为Excel(.xlsx)，列头与页面表格一致。
- **下钻**：图表中支持点击数据点或扇区，弹出对应的明细列表窗口或跳转至带筛选条件的列表页。
- **空状态**：无数据时显示“暂无数据”占位图。
- **加载状态**：图表和表格加载时显示骨架屏或Spin加载动画。

## 4. 非功能需求

- **性能**：所有统计图表查询API响应时间在95%分位下≤3秒，复杂事故分析≤10秒。
- **数据刷新**：实时数据查询列表支持10秒自动刷新，可手动暂停。
- **权限**：按角色控制数据可见范围（政府全局，企业仅自身数据）。
- **日志**：报告生成和导出操作记录审计日志。
