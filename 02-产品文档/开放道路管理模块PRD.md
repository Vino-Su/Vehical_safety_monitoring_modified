# 名词解释

| 名词/缩写   | 解释                                                          |
| ------- | ----------------------------------------------------------- |
| RSU     | Road Side Unit，路侧单元，部署在道路沿线的通信设备，用于车路通信                     |
| 自动驾驶等级  | 参照国家标准 GB/T 40429-2021，分为 L2（组合驾驶辅助）、L3（有条件自动驾驶）、L4（高度自动驾驶） |
| 差异化准入配置 | 针对同一道路，按不同自动驾驶等级和业务类型分别配置准入规则，实现道路资源的精细化分配                  |
| 开放状态    | 道路当前的开放管控状态，枚举值：开放、暂停、关闭                                    |
| 业务类型    | 智能网联车辆在道路上开展的业务类别，枚举值：载人测试、载货测试、环卫作业、物流配送、示范运营              |
| 开放等级    | 评估模型输出的道路开放建议等级，枚举值：完全开放、条件开放、限制开放、禁止开放                     |
| 融合模型    | 综合复杂度模型、风险模型、场景适配模型的多模型融合决策引擎                               |

## 1 业务背景与使用角色

### 1.1 业务背景
- 模块定位：开放道路资源管理模块是智能网联汽车安全监测平台的三大核心业务模块之一，定位为智能网联汽车道路测试与示范应用的基础资源保障模块，构建 " 资源目录化、管理精细化、查询便捷化 " 的开放道路资源管理体系
- 核心特点：以道路为维度，覆盖 " 建档→状态管控→可视化→评估→统计 " 的全生命周期管理；支持差异化准入配置实现道路资源的精细化分配；内置多模型融合评估引擎，支持评估结果的版本化管理和决策追溯
- 当前痛点：针对襄阳市智能网联开放道路资源分散、管理缺乏标准化的现状，实现全市范围内符合测试条件的城市道路统一资源化、标准化、可视化管理
- 关联模块关系：与测试示范监管模块（monitor/）形成联动——道路资源数据为测试审批提供合规基础设施依据，道路状态变更联动测试任务终止，评估结果反哺道路资源动态调整；与监测数据分析模块（data-analysis/）共享道路基础数据

### 1.2 使用角色

| 角色               | 该模块中的职责                                        | 客户端   |
| ---------------- | ---------------------------------------------- | ----- |
| 政府监管部门（经信/交通/公安） | 道路信息建档与编辑，道路状态变更管控，差异化准入配置管理，评估指标配置，全模块数据查看与导出 | Web 端 |
| 产业运营主体（主机厂/运营企业） | 查看已开放道路信息（地图可视化、统计查询），查看评估结果，查询可用道路资源辅助测试路线申请  | Web 端 |
| 第三方服务机构          | 查看道路资源信息、评估结果、评估日志，获取道路资源数据以支撑技术评估             | Web 端 |

### 1.3 核心目标与价值

| 目标维度   | 说明                                             |
| ------ | ---------------------------------------------- |
| 统一资源管理 | 建立全市开放测试道路的统一信息台账，确保道路数据的完整性、一致性和可追溯性          |
| 精细化管控  | 支持道路状态动态调整（开放/暂停/关闭/恢复），实现按自动驾驶等级和业务类型的差异化准入配置 |
| 空间可视化  | 通过地图多图层直观展示道路分布与评估结果专题图，辅助空间决策分析               |
| 科学评估   | 内置多模型融合评估引擎，支持复杂度、风险、场景适配三维度评估，输出科学开放等级建议      |
| 统计决策   | 按区域维度聚合统计道路开放情况并支持可视化图表，支撑管理部门宏观决策             |

#### 3.1.6 交互流程

**新建道路：**

1. 用户点击 " 新建道路 " → 打开新建道路弹窗，表单为空，道路编号自动生成并置灰显示
2. 用户填写表单 → 点击 " 确定 "
   - 校验成功 → 保存数据 + Toast 提示 " 新建成功 " + 关闭弹窗 + 列表刷新
   - 校验失败 → 在对应字段下方显示红色错误提示，弹窗不关闭
3. 点击 " 取消 " → 关闭弹窗，不保存数据

**编辑道路：**

1. 用户点击 " 编辑 " → 打开编辑道路弹窗，表单预填当前数据，道路编号置灰不可修改
2. 用户修改表单 → 点击 " 确定 "
   - 校验成功 → 保存数据 + Toast 提示 " 编辑成功 " + 关闭弹窗 + 列表刷新
   - 校验失败 → 在对应字段下方显示红色错误提示，弹窗不关闭
3. 点击 " 取消 " → 关闭弹窗，不保存修改

**查看详情：**

1. 用户点击 " 详情 " → 打开道路详情弹窗（只读），按分组展示全部字段：基础标识信息、空间属性信息、交通运行信息、基础设施信息、测试场景信息、开放管控信息、更新记录信息
2. 底部仅显示 " 关闭 " 按钮

**删除操作：**

1. 用户点击 " 删除 " → 弹出二次确认弹窗（见附录 A.2），确认内容：" 确定删除该道路信息吗？删除后数据不可恢复。"
2. 若该道路存在进行中的关联申请流程 → 弹窗提示 " 该道路存在正在进行的关联申请流程，无法删除 "，阻断操作
3. 点击 " 确定 " → 逻辑删除 + 记录删除日志 + Toast 提示 " 删除成功 " + 列表刷新
4. 点击 " 取消 " → 关闭弹窗，不执行删除

**导出操作：**

1. 用户点击 " 导出 " → 导出规则见附录 A.3
2. 导出格式：.xlsx，文件命名：道路信息建档 _YYYYMMDD.xlsx
3. 导出内容：导出的字段与导入的模板字段相同

## 4 业务规则汇总

### 4.1 唯一性规则
- 道路编号全局唯一
- 同一道路下，差异化准入配置的 " 自动驾驶等级 + 业务类型 " 组合不可重复
- 同一道路在同一评估时间点的评估结果唯一（同一版本仅有一条有效记录）

### 4.2 删除规则
- 删除方式：逻辑删除（标记删除状态），不物理删除数据
- 删除权限：仅政府监管部门和第三方服务机构可执行
- 删除确认：弹出二次确认弹窗（见附录 A.2）
- 前置校验：删除道路前校验是否存在进行中的关联申请流程，存在则阻断并提示
- 删除后行为：Toast 提示 " 删除成功 "，列表刷新，记录删除操作日志

### 4.3 状态流转

道路开放状态流转规则：

```
开放 ──暂停──▶ 暂停
开放 ──关闭──▶ 关闭
暂停 ──恢复──▶ 开放
暂停 ──关闭──▶ 关闭
关闭 ──恢复──▶ 开放
```

- 各状态变更操作必须填写变更原因
- 暂停/关闭操作前，校验是否存在进行中的关联申请流程，存在则弹出警告但允许继续，确认后自动向测试示范监管模块发送任务终止指令
- 状态变更后，记录至 " 状态变更记录 "Tab
- 暂停/关闭状态的道路应在测试示范准入申请的道路选择列表中自动过滤

### 4.4 字段联动规则

| 触发字段 | 触发条件 | 联动行为 |
| -------- | -------- | -------- |
| 开放状态（道路建档表单） | 值为 " 关闭 " | 开放时段 - 开始时间、开放时段 - 结束时间置灰不可填 |
| 开放状态（道路建档表单） | 值为 " 开放 " 或 " 暂停 " | 开放时段 - 开始时间、开放时段 - 结束时间恢复可填且必填 |

### 4.5 计算规则

**综合得分计算**：综合得分 = (100 - 复杂度指数 × 0.33 - 风险指数 × 0.34 + 场景适配得分 × 0.33 + 20) ，结果截断至 [0, 100] 区间，保留 1 位小数。

**开放等级映射**：

- 综合得分 ≥ 85 → 完全开放
- 综合得分 ≥ 70 → 条件开放
- 综合得分 ≥ 50 → 限制开放
- 综合得分 < 50 → 禁止开放

**融合规则优先级**：规则按优先级从高到低（1 为最高）依次匹配，首次命中即输出结果，不继续匹配后续规则。全局禁止条件（风险=高→禁止开放）优先级最高。
