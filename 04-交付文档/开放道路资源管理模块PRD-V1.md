# 名词解释

| 名词/缩写   | 解释                                                          |
| ------- | ----------------------------------------------------------- |
| RSU     | Road Side Unit，路侧单元，部署在道路沿线的通信设备，用于车路通信                     |
| 自动驾驶等级  | 参照国家标准 GB/T 40429-2021，分为 L2（组合驾驶辅助）、L3（有条件自动驾驶）、L4（高度自动驾驶） |
| 差异化准入配置 | 针对同一道路，按不同自动驾驶等级和业务类型分别配置准入规则，实现道路资源的精细化分配                  |
| 开放状态    | 道路当前的开放管控状态，枚举值：开放、暂停、关闭                                    |
| 业务类型    | 智能网联车辆在道路上开展的业务类别，枚举值：载人测试、载货测试、环卫作业、物流配送、示范运营              |
| 开放等级    | 评估模型输出的道路开放建议等级，枚举值：完全开放、条件开放、限制开放、禁止开放                     |
| 融合模型    | 综合复杂度模型、风险模型、场景适配模型的多模型融合决策引擎                               |
| 襄阳市行政区划 | 襄城区‌、樊城区‌、襄州区、南漳县‌、谷城县‌、保康县‌、枣阳市‌、宜城市‌、老河口市、高新区、东津新区、鱼梁洲开发区 |

## 1 背景

 开放道路资源管理模块是智能网联汽车安全监测平台的三大核心业务模块之一，定位为智能网联汽车道路测试与示范应用的基础资源保障模块，构建 " 资源目录化、管理精细化、查询便捷化 " 的开放道路资源管理体系

以道路为维度，覆盖 " 建档→状态管控→可视化→评估→统计 " 的全生命周期管理；支持差异化准入配置实现道路资源的精细化分配；内置多模型融合评估引擎，支持评估结果的版本化管理和决策追溯

关联模块关系：与测试示范监管模块形成联动——道路资源数据为测试审批提供合规基础设施依据，道路状态变更联动测试任务终止，评估结果反哺道路资源动态调整；与监测数据分析模块共享道路基础数据

#### 6.1.4 新建道路弹窗

1. 用户点击 " 新建道路 " → 打开新建道路弹窗，表单为空，道路编号自动生成并置灰显示
2. 用户填写表单 → 点击 " 确定 "
   - 校验成功 → 保存数据 + Toast 提示 " 新建成功 " + 关闭弹窗 + 列表刷新
   - 校验失败 → 在对应字段下方显示红色错误提示，弹窗不关闭
3. 点击 " 取消 " → 关闭弹窗，不保存数据

#### 6.1.5 编辑道路弹窗

1. 用户点击 " 编辑 " → 打开编辑道路弹窗，表单预填当前数据，道路编号置灰不可修改
2. 用户修改表单 → 点击 " 确定 "
   - 校验成功 → 保存数据 + Toast 提示 " 编辑成功 " + 关闭弹窗 + 列表刷新
   - 校验失败 → 在对应字段下方显示红色错误提示，弹窗不关闭
1. 点击 " 取消 " → 关闭弹窗，不保存修改
#### 6.1.6 批量导入弹窗

1. 用户点击 " 批量导入 " → 打开批量导入弹窗
2. 弹窗内容：下载导入模板按钮、文件拖拽上传区域（支持 Excel/CSV，单次最大 1000 条）、注意事项说明
3. 必填字段：道路名称、所属行政区域、道路等级、适用测试场景、开放状态、开放时段、适用自动驾驶等级、适用业务类型
4. 点击 " 确认导入 " → 数据校验（必填项、格式、枚举值合法性） → 通过校验 + Toast 提示导入结果；若存在未通过校验的数据，则 toast 提示导入失败，给出失败原因，不执行本次导入
5. 点击 " 取消 " → 关闭弹窗，不执行导入

### 6.3 F03 地图可视化展示

#### 6.3.1 地图区域

##### 6.3.1.1 专题图层切换

 图层切换按钮位于地图左上角，共有 5 个主题图层，默认显示基础信息图层

地图区域默认加载全市已录入的道路矢量资源，并根据所处的不同专题图层，显示不同的颜色

| 图层     | 颜色映射规则                             | 数据来源         |
| ------ | ---------------------------------- | ------------ |

-  **版本号生成规则：** 版本号由【主版本号 . 次版本号】组成，其中主版本号由后端开发人员控制，一般仅在算法有重大变更时更新，默认为1；次版本号则在每次保存配置时，自动加1
- 点击 " 重置为默认 "，弹出重置确认弹窗，点击确认重置后，各 Tab 参数恢复默认值 + Toast 提示 " 已重置为默认配置 "

# 附录 统一组件规范

## 1 A.1 分页组件

| 配置项    | 规则                 |
| ------ | ------------------ |
| 每页条数选项 | 10、20、50、100，默认 10 |
| 显示信息   | 总记录数、当前页码/总页数      |
| 翻页控件   | 首页、上一页、页码、下一页、末页   |
| 切换每页条数 | 下拉选择，切换后从第一页重新加载   |

## 2 A.2 删除确认组件
1. 用户点击 " 删除 " → 弹出二次确认弹窗
2. 确认弹窗内容：" 确定删除该信息吗？删除后数据不可恢复。"
3. 点击 " 确定 " → 执行逻辑删除 + 记录删除操作日志 + 列表刷新
4. 点击 " 取消 " → 关闭确认弹窗，不执行删除

**删除方式：** 逻辑删除（标记删除状态），不物理删除数据。

## 3 A.3 导出组件

| 触发条件      | 行为                                           |
| --------- | -------------------------------------------- |
| 用户选中了特定数据 | 导出用户选择的数据，不支持跨页导出                            |
| 用户未选择特定数据 | 弹出提示：" 是否导出所有数据？"，用户选择 " 是 " 则导出当前筛选条件下的所有数据 |

**导出格式：** .xlsx

**文件命名规则：** {模块名称}_YYYYMMDD.xlsx

## 4 A.4 附件上传组件

| 规则项     | 说明                          |
| ------- | --------------------------- |
| 支持格式    | Excel(.xlsx/.xls)、CSV(.csv) |
| 单文件大小限制 | ≤20MB                       |
| 批量导入上限  | 单次最大 1000 条                   |

## 5 A.5 复选框组件
- 表头存在全选复选框，勾选后选中当前页所有行
- 支持单行勾选
- 勾选状态用于导出、批量操作等场景
- 全选仅作用于当前页数据，不支持跨页全选
