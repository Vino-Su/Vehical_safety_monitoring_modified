## 1 道路评估模型

### 1.1 原始数据

  道路基础数据：道路信息

  事故数据：监测数据分析 - 事故模块

  交通事件/流量数据：后期外部对接，本期可手动标注或缺省处理

### 1.2 复杂度模型计算规则

复杂度模型分为车道复杂度、交叉口密度、混合交通比例、坡度/曲率、信号灯完整度等几个维度进行评价。

各指标的权重支持在系统中手动配置（5 项权重之和必须等于 1。），默认值如下：

| 指标名称   | 默认权重 | 取值范围 |
| ------ | ---- | ---- |
| 车道复杂度  | 0.25 | 0-1  |
| 交叉口密度  | 0.25 | 0-1  |
| 混合交通比例 | 0.20 | 0-1  |
| 坡度/曲率  | 0.15 | 0-1  |
| 信号灯完整度 | 0.15 | 0-1  |

具体的各指标需要根据采集的道路参数进行标准化得分，其中根据指标的类型不同采用不同的积分方式，参见下表

**分类/二值指标：采用固定分值表**

此类指标输入为离散类别或是否判断，直接映射为固定得分。

| 指标     | 输入条件    | 固定得分 | 业务解释                 |
| ------ | ------- | ---- | -------------------- |
| 车道复杂度  | 车道数 ≤ 2 | 90   | 车道少，通行空间受限，自动驾驶避让难度大 |
|        | 车道数 3-4 | 60   |                      |
|        | 车道数 ≥ 5 | 20   | 车道多，通行规则相对清晰         |
| 混合交通比例 | 存在机非混行  | 90   | 机非混行增加感知与预测难度        |
|        | 无非混行    | 20   |                      |

**连续数值指标：采用分段线性函数**

此类指标输入为连续数值，按分段线性函数计算，保证相邻区间平滑过渡。

| 指标     | 输入变量            | 得分公式                                                                                                                  | 说明                           |
| ------ | --------------- | --------------------------------------------------------------------------------------------------------------------- | ---------------------------- |
| 交叉口密度  | 每公里交叉口数 `d`     | `d ≥ 5` 时：score = 90 + (d-5) × 2，上限 100<br>`2 ≤ d < 5` 时：score = 40 + (d-2) × 16.67<br>`d < 2` 时：score = 20 + d × 10  | 交叉口越密，复杂度越高                  |
| 坡度/曲率  | 最大坡度 `s`（%）     | `s ≥ 8` 时：score = 90 + (s-8) × 1.25，上限 100<br>`4 ≤ s < 8` 时：score = 40 + (s-4) × 12.5<br>`s < 4` 时：score = 20 + s × 5 | 坡度越大、曲率越急，复杂度越高              |
| 信号灯完整度 | 信号灯覆盖率 `c`（0-1） | score = (1 - c) × 100                                                                                                 | 覆盖率越低，复杂度越高；完全覆盖为 0，无覆盖为 100 |

说明：

> 1. 固定分值表和线性函数的分段断点可在页面中配置，以上默认业务规则。
> 2. 当指标输入数据缺失时，该指标按 60 分计算，并在评估日志中标注「指标数据缺失，采用默认中分」。

**复杂度指数计算**

```
复杂度指数 = Σ（指标标准化得分 × 指标权重）
```

结果保留一位小数

**复杂度等级映射**

| 等级   | 阈值              |
| ---- | --------------- |
| 低复杂度 | 复杂度指数 < 40      |
| 中复杂度 | 40 ≤ 复杂度指数 ≤ 69 |
| 高复杂度 | 复杂度指数 ≥ 70      |

### 1.3 风险度计算规则

风险度根据事故频率、事故严重度、交通事件频率、流量修正系数等参数进行计算，具体各指标的权重默认值见下表，支持手动配置（4 项权重之和必须等于 1）

| 指标名称   | 默认权重 | 取值范围 |
| ------ | ---- | ---- |
| 事故频率   | 0.35 | 0-1  |
| 事故严重度  | 0.35 | 0-1  |
| 交通事件频率 | 0.20 | 0-1  |
| 流量修正系数 | 0.10 | 0-1  |

其中，相关事件的数据时间窗口，默认采用近 24 个月的历史数据，但在计算「事故频率」时引入时间衰减加权，越近期的事故对风险得分影响越大。

| 时间区间 (月) | 衰减权重 | 说明       |
| ------- | ---- | -------- |
| [0,6]   | 1.0  | 事故影响最大   |
| （6,12]  | 0.7  | 事故影响中等   |
| （12,24] | 0.4  | 事故影响较小   |
| >24     | 0.2  | 仅作长期趋势参考 |

**时间衰减后事故数计算公式**：

```
n_decayed = Σ(各事故 × 对应时间区间衰减权重)
```

> 示例：某道路近 6 个月 2 起、6-12 个月 1 起、12-24 个月 1 起，则 n_decayed = 2×1.0 + 1×0.7 + 1×0.4 = 3.1。

统计窗口与衰减权重可在 F06 配置，默认采用上表。

具体的各指标需要根据采集的道路参数进行标准化得分，其中根据指标的类型不同采用不同的积分方式，参见下表

分类/二值指标：采用固定分值表

| 指标     | 输入条件                    | 固定得分 | 业务解释       |
| ------ | ----------------------- | ---- | ---------- |
| 事故严重度  | 高（伤亡 ≥ 1 人或财产损失 ≥ 10 万） | 90   | 事故后果严重     |
|        | 中（轻微伤或财产损失 1-10 万）      | 60   |            |
|        | 低（无伤亡、财产损失 < 1 万）       | 20   |            |
| 交通事件频率 | 频发（月均 ≥ 3 次）            | 90   | 违章、拥堵等事件频繁 |
|        | 偶发（月均 1-2 次）            | 60   |            |
|        | 极少（月均 < 1 次）            | 20   |            |
| 流量修正系数 | 交通流量等级「高」               | 90   | 流量大，风险叠加   |
|        | 交通流量等级「中」               | 60   |            |
|        | 交通流量等级「低」               | 20   |            |
|        | 无数据                     | 60   | 数据缺失按中分处理  |

连续数值指标：采用分段线性函数

| 指标   | 输入变量                 | 得分公式                                                                                                                                                                 | 说明              |
| ---- | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------- |
| 事故频率 | 时间衰减后事故数 `n_decayed` | `n_decayed ≥ 5` 时：score = 90 + (n_decayed-5) × 2，上限 100<br>`2 ≤ n_decayed < 5` 时：score = 40 + (n_decayed-2) × 16.67<br>`n_decayed < 2` 时：score = 20 + n_decayed × 10 | 事故次数越多、越近期，风险越高 |

>固定分值表和线性函数的分段断点可在页面中配置，以上默认业务规则。

**风险指数计算**

```
风险指数 = Σ（指标标准化得分 × 指标权重）
```

结果保留 1 位小数。

**风险等级阈值**

| 等级  | 阈值             |
| --- | -------------- |
| 低风险 | 风险指数 < 40      |
| 中风险 | 40 ≤ 风险指数 ≤ 69 |
| 高风险 | 风险指数 ≥ 70      |

### 1.4 场景适配模型

**场景列表及映射条件**

| 场景名称 | 最低车道数 | 允许机非混行 | 最低照明等级 | 允许最大坡度 |
| ---- | ----- | ------ | ------ | ------ |
| 城区道路 | 4     | 否      | 2      | 8%     |
| 园区道路 | 2     | 是      | 1      | 12%    |
| 环卫作业 | 2     | 是      | 1      | 10%    |
| 物流配送 | 2     | 否      | 2      | 8%     |
| 公交运营 | 4     | 否      | 3      | 6%     |

每个场景从 4 个维度打分，每个维度满分 25 分：

| 维度   | 得分规则                                                             |
| ---- | ---------------------------------------------------------------- |
| 车道数  | 实际车道数 ≥ 最低车道数：25 分；否则 0 分                                        |
| 机非混行 | 若道路存在机非混行，且场景「允许机非混行」为「是」：25 分；道路不存在机非混行：25 分；道路存在机非混行但场景不允许：0 分 |
| 照明等级 | 实际照明等级 ≥ 最低照明等级：25 分；否则 0 分                                      |
| 最大坡度 | 实际最大坡度 ≤ 允许最大坡度：25 分；否则 0 分                                      |

```
场景适配度得分 = Σ 各维度得分
```

结果范围为 0-100 分，保留 1 位小数。

**适配状态判定**

| 状态  | 条件                      |
| --- | ----------------------- |
| 适用  | 场景适配度得分 ≥ 适配阈值（默认 60 分） |
| 不适用 | 场景适配度得分 < 适配阈值          |

**最佳适配场景**

取所有场景中「场景适配度得分最高且状态为适用」的场景作为「最佳适配场景」，若得分相同，则最佳适配场景可以为多个。

若所有场景均不适用，则「最佳适配场景」显示为「无」

### 1.5 综合得分计算

**计算公式**

```
综合得分 = (100 - 复杂度指数) × 复杂度权重
       + (100 - 风险指数) × 风险权重
       + 场景适配度得分 × 场景适配权重
```

其中：

- 复杂度权重、风险权重、场景适配权重取自页面子模型权重配置；
- 三项权重之和必须等于 1；
- 结果截断到 [0, 100]，保留 1 位小数。
**默认权重**

| 子模型    | 默认权重 |
| ------ | ---- |
| 复杂度模型  | 0.33 |
| 风险模型   | 0.34 |
| 场景适配模型 | 0.33 |

支持手动配置

**综合得分与开放等级映射**

| 建议开放等级 | 综合得分阈值         |
| ------ | -------------- |
| 完全开放   | 综合得分 ≥ 85      |
| 条件开放   | 70 ≤ 综合得分 < 85 |
| 限制开放   | 50 ≤ 综合得分 < 70 |
| 禁止开放   | 综合得分 < 50      |

### 1.6 融合规则与开放等级判定
#### 1.6.1 判定顺序

采用「准入条件前置拦截、融合规则优先、综合得分映射兜底」的三段式流程：

```
 1. 第一步：硬性准入条件检查
   - 检查道路是否满足最低安全准入条件（见 7.2）；
   - 任一条件不满足，直接输出「建议开放等级 = 禁止开放」，并记录未满足条件；
   - 不再进入后续子模型计算与融合规则匹配。

2. 第二步：子模型计算
   - 计算复杂度指数/等级、风险指数/等级、各场景适配度得分/状态。

3. 第三步：融合规则匹配
   - 按优先级从高到低依次匹配 F06 中启用的融合规则；
   - 首次命中即输出「建议开放等级」，不再继续匹配；
   - 全局禁止条件「风险等级 = 高 → 禁止开放」作为最高优先级隐式规则。

4. 第四步：综合得分映射兜底
   - 若未命中任何融合规则，则使用 6.4 的映射表由综合得分确定「建议开放等级」。

5. 第五步：业务规则映射
   - 根据确定的「建议开放等级」，从映射关系表（见 8）中获取适用自动驾驶等级、允许测试类型、限制时段、管控说明。
```

#### 1.6.2 硬性准入条件

以下任一条件不满足，该道路直接判定为「禁止开放」，不再进入后续子模型计算：

| 准入条件       | 阈值      | 数据来源 | 缺失处理               |
| ---------- | ------- | ---- | ------------------ |
| 最小车道宽度     | ≥ 3.0 m | 道路信息 | 数据缺失时，该条件视为「满足」    |
| 最大坡度       | ≤ 6%    | 道路信息 | 数据缺失时，该条件视为「满足」    |
| 交通信号覆盖率    | ≥ 80%   |      | 覆盖率数据缺失时，该条件视为「满足」 |
| 视距严重不足路段占比 | < 10%   | 道路信息 | 数据缺失时，该条件视为「满足」    |

以上硬性准入条件，可在页面中进行配置

#### 1.6.3 默认融合规则

融合规则的核心价值在于处理**综合得分公式无法表达或不应由公式表达**的外部条件、政策法规、特殊事件和动态状态。

| 优先级 | 条件描述                 | 动作   | 默认状态 |
| --- | -------------------- | ---- | ---- |
| 1   | 道路类型 = 高速公路/城市快速路    | 禁止开放 | 启用   |
| 2   | 风险等级 = 高             | 禁止开放 | 启用   |
| 3   | 复杂度 = 高 AND 风险 = 高   | 限制开放 | 启用   |
| 4   | 风险 = 中 AND 场景适配 < 60 | 限制开放 | 禁用   |

该规则不支持编辑和增删，支持启用/通用的配置

### 1.7 开放等级业务规则映射
#### 1.7.1 开放等级映射表

| 建议开放等级 | 适用自动驾驶等级     | 允许测试类型      | 限制时段        | 管控要求      |
| ------ | ------------ | ----------- | ----------- | --------- |

## 2 评估触发与版本管理
#### 2.1.1 触发方式

| 触发类型   | 触发条件               | 评估范围         |
| ------ | ------------------ | ------------ |
| 定期评估   | 每日凌晨 2:00 系统自动执行   | 全量道路（含已关闭道路） |
| 道路信息变更 | 道路建档/编辑保存后         | 单条道路         |
| 道路状态变更 | 道路状态从「暂停」恢复为「开放」时  | 单条道路         |
| 手动触发   | 用户在 F05 点击「手动触发评估」 | 全量道路（含已关闭道路） |
| 重新评估   | 用户在 F05 点击「重新评估」   | 单条道路         |

#### 2.1.2 配置版本绑定

- 每次保存 F06 评估指标配置，自动生成新版本号（如 V3.3）；
- 每次评估执行时，记录当前生效的配置版本号；
- 评估结果表字段 `config_version` 必填；
- 评估日志中保存「配置快照」：复杂度模型权重、风险模型权重、场景适配模型权重、硬性准入条件阈值、融合规则列表、映射关系表、限制条件模板库、各阈值。
#### 2.1.3 历史版本对比

F05「评估记录对比」选择两个版本后，展示：

- 综合得分变化；
- 复杂度指数变化；
- 风险指数变化；
- 场景适配得分变化；
- 开放等级变化；
- 命中融合规则变化（新增/消失/动作变化）。

#### 2.1.4 评估结果输出格式

每条道路的评估结果除基础字段外，还需输出以下结构化数据，供 F05 详情页、F07 日志详情页及地图可视化使用。

##### 2.1.4.1 判定依据说明生成

评估结果需自动生成结构化判定依据，用于详情页展示、日志留痕及对外报告导出。判定依据包含以下要素：

| 要素       | 内容                                                                           |
| -------- | ---------------------------------------------------------------------------- |
| 准入条件检查结果 | 是否通过，未通过项及实际值                                                                |
| 复杂度评分及等级 | 复杂度指数、等级、各子指标贡献                                                              |
| 风险评分及等级  | 风险指数、等级、事故时间衰减说明、各子指标贡献                                                      |
| 场景适配排序   | 各场景适配度得分及状态，最佳适配场景                                                           |
| 命中规则     | 融合的优先级、条件、动作、规则来源（政策法规/特殊区域/动态状态/外部预警/子模型组合）；若命中外部条件，记录触发数据值；若未命中则说明采用综合得分映射 |
| 综合得分     | 计算公式、各子模型转换后得分及权重                                                            |
| 映射结果     | 开放等级、适用自动驾驶等级、允许测试类型、限制时段                                                    |
| 限制条件来源   | 自动触发的限制条件及触发原因                                                               |
| 数据缺失说明   | 哪些指标采用默认值，默认原因                                                               |
