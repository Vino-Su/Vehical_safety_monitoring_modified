## 1. 方案目标与范围

### 1.1 目标
明确开放道路评估模块的后台数据计算链路，使「评估结果管理、评估指标配置、评估日志管理」三个页面在数据口径、计算逻辑、结果解释上保持一致，并具备可落地性。

### 1.2 范围
覆盖 PRD 中 F05（评估结果管理）、F06（评估指标配置）、F07（评估日志管理）及 F03（地图可视化评估专题图层）所依赖的评估数据计算规则。

不覆盖：前端交互细节、权限控制、UI 样式。

## 3. 计算流程总览

```
原始数据输入
  ├── 道路基础数据（F01 道路建档 + 补充数据字段）
  ├── 事故数据（监测数据分析-事故模块）
  └── 交通事件/流量数据（后期外部对接，本期可手动标注或缺省处理）
         ↓
硬性准入条件检查
  ├── 任一条件不满足 → 建议开放等级 = 禁止开放
  └── 全部满足 → 进入子模型计算
         ↓
子模型计算
  ├── 复杂度模型 → 复杂度指数（0-100）+ 复杂度等级（低/中/高）
  ├── 风险模型 → 风险指数（0-100）+ 风险等级（低/中/高）
  └── 场景适配模型 → 各场景适配度得分（0-100）+ 适配状态（适用/不适用）
         ↓
融合决策
  ├── 按融合规则优先级首次命中 → 建议开放等级
  └── 未命中任何规则 → 按综合得分映射 → 建议开放等级
         ↓
业务规则映射
  └── 根据建议开放等级 → 适用自动驾驶等级、允许测试类型、限制时段、管控说明
         ↓
结果输出与归档
  ├── 评估结果记录（含配置版本号、各子模型得分、命中规则、雷达图数据、判定依据）
  ├── 评估日志记录（配置快照、数据来源、计算过程）
  └── 反哺地图可视化专题图层
```

## 4. 原始数据输入

### 4.1 道路基础数据（来自 F01 道路信息建档）

| 数据项 | 用途 |
|--------|------|
| 车道数 | 复杂度模型「车道复杂度」、场景适配模型「最低车道数」判定 |
| 车道类型 | 辅助判断机非混行情形 |
| 路段长度 | 交叉口密度、事故频率等归一化计算 |
| 区域轮廓散点 | 后期对接高精地图，用于坡度/曲率/交叉口密度自动计算 |
| 交通流量等级 | 风险模型「流量修正系数」判定 |
| RSU 配置数量、感知设备类型/数量 | 场景适配与基础设施覆盖辅助参考（本期不计入核心公式） |
| 适用测试场景 | 场景适配模型候选场景 |

### 4.2 补充数据字段（建议新增至 F01 或允许批量导入）

| 字段名称     | 字段类型    | 是否必填 | 用途              |
| -------- | ------- | ---- | --------------- |
| 交叉口数量    | 整数      | 否    | 计算交叉口密度         |
| 是否存在机非混行 | 枚举（是/否） | 否    | 场景适配模型判定        |
| 照明等级     | 整数（1-5） | 否    | 场景适配模型判定        |
| 最大坡度     | 百分比数值   | 否    | 场景适配模型判定、硬性准入条件检查 |
| 最小车道宽度   | 数值（米）   | 否    | 硬性准入条件检查        |
| 视距不足路段占比 | 百分比数值   | 否    | 硬性准入条件检查        |
| 信号灯数量    | 整数      | 否    | 复杂度模型「信号灯完整度」判定 |

> 说明：上述字段本期允许为空，为空时按默认值或提示「数据缺失，评估结果供参考」。

### 4.3 事故数据（来自监测数据分析-事故模块）

| 数据项          | 用途             |
| ------------ | -------------- |
| 道路/路段维度的事故次数 | 风险模型「事故频率」     |
| 事故伤亡人数、财产损失  | 风险模型「事故严重度」    |
| 事故时间         | 用于筛选近 N 个月统计窗口 |

### 4.4 交通事件/流量数据（本期规划）

| 数据项 | 用途 |
|--------|------|
| 违章次数、拥堵事件次数 | 风险模型「交通事件频率」 |
| 实时/历史流量数据 | 风险模型「流量修正系数」 |

本期若无外部数据，「交通事件频率」和「流量修正系数」可默认给中间分（如 50 分），并在评估日志中标注「数据缺失，采用默认值」。

### 4.5 融合规则外部数据

以下数据用于支撑融合规则中不可由子模型得分表达的政策法规、特殊区域、动态状态类条件：

| 数据项 | 用途 | 数据来源 | 本期处理建议 |
|--------|------|----------|--------------|
| 道路类型 | 判断是否为高速公路/城市快速路 | F01 道路建档 | 必填 |
| 道路状态 | 判断是否施工中/封路 | F01 道路状态管理 | 必填 |
| 特殊区域标识 | 判断道路是否位于学校/医院/政府周边 | 地图 POI 服务或人工标注 | 本期可人工标注，二期对接地图服务 |
| 近 3 个月死亡事故记录 | 一票否决条件触发 | 监测数据分析-事故模块 | 从事故数据按伤亡维度筛选 |
| 气象预警等级 | 红色预警时临时禁止开放 | 气象部门接口/手动录入 | 本期可手动录入，二期对接气象接口 |

> 说明：上述外部数据本期允许缺失，缺失时对应融合规则视为「未命中」，不影响其他规则匹配和综合得分映射。

## 5. 子模型计算规则

### 5.1 复杂度模型

#### 5.1.1 指标权重（与 F06 保持一致，可配置）

| 指标名称   | 默认权重 | 取值范围 |
| ------ | ---- | ---- |
| 车道复杂度  | 0.25 | 0-1  |
| 交叉口密度  | 0.25 | 0-1  |
| 混合交通比例 | 0.20 | 0-1  |
| 坡度/曲率  | 0.15 | 0-1  |
| 信号灯完整度 | 0.15 | 0-1  |

**约束**：5 项权重之和必须等于 1。

#### 5.1.2 各指标标准化得分（0-100，越高越复杂）

为避免「区间给分」无法落地计算的问题，复杂度子指标按输入类型分为两类：

**A. 分类/二值指标：采用固定分值表**

此类指标输入为离散类别或是否判断，直接映射为固定得分。

| 指标     | 输入条件    | 固定得分 | 业务解释                 |
| ------ | ------- | ---- | -------------------- |
| 车道复杂度  | 车道数 ≤ 2 | 90   | 车道少，通行空间受限，自动驾驶避让难度大 |
|        | 车道数 3-4 | 60   |                      |
|        | 车道数 ≥ 5 | 20   | 车道多，通行规则相对清晰         |
| 混合交通比例 | 存在机非混行  | 90   | 机非混行增加感知与预测难度        |
|        | 无非混行    | 20   |                      |

**B. 连续数值指标：采用分段线性函数**

此类指标输入为连续数值，按分段线性函数计算，保证相邻区间平滑过渡。

| 指标     | 输入变量            | 得分公式                                                                                                                  | 说明                           |
| ------ | --------------- | --------------------------------------------------------------------------------------------------------------------- | ---------------------------- |
| 交叉口密度  | 每公里交叉口数 `d`     | `d ≥ 5` 时：score = 90 + (d-5) × 2，上限 100<br>`2 ≤ d < 5` 时：score = 40 + (d-2) × 16.67<br>`d < 2` 时：score = 20 + d × 10  | 交叉口越密，复杂度越高                  |
| 坡度/曲率  | 最大坡度 `s`（%）     | `s ≥ 8` 时：score = 90 + (s-8) × 1.25，上限 100<br>`4 ≤ s < 8` 时：score = 40 + (s-4) × 12.5<br>`s < 4` 时：score = 20 + s × 5 | 坡度越大、曲率越急，复杂度越高              |
| 信号灯完整度 | 信号灯覆盖率 `c`（0-1） | score = (1 - c) × 100                                                                                                 | 覆盖率越低，复杂度越高；完全覆盖为 0，无覆盖为 100 |

> 说明：
> 1. 固定分值表和线性函数的分段断点可在 F06 中配置，本方案先给出默认业务规则。
> 2. 当指标输入数据缺失时，该指标按 60 分计算，并在评估日志中标注「指标数据缺失，采用默认中分」。

#### 5.1.3 复杂度指数

```
复杂度指数 = Σ（指标标准化得分 × 指标权重）
```

结果保留 1 位小数。

#### 5.1.4 复杂度等级阈值（F06 为准）

| 等级   | 阈值              |
| ---- | --------------- |
| 低复杂度 | 复杂度指数 < 40      |
| 中复杂度 | 40 ≤ 复杂度指数 ≤ 69 |
| 高复杂度 | 复杂度指数 ≥ 70      |

> 地图可视化 F03 的复杂度专题图层颜色阈值，同步采用本表。

### 5.2 风险模型

#### 5.2.1 指标权重（与 F06 保持一致，可配置）

| 指标名称   | 默认权重 | 取值范围 |
| ------ | ---- | ---- |
| 事故频率   | 0.35 | 0-1  |
| 事故严重度  | 0.35 | 0-1  |
| 交通事件频率 | 0.20 | 0-1  |
| 流量修正系数 | 0.10 | 0-1  |

**约束**：4 项权重之和必须等于 1。

#### 5.2.2 时间窗与时间衰减加权

默认使用近 24 个月历史数据，但在计算「事故频率」时引入时间衰减加权，越近期的事故对风险得分影响越大。

| 时间区间     | 衰减权重 | 说明       |
| -------- | ---- | -------- |
| 近 6 个月   | 1.0  | 事故影响最大   |
| 6-12 个月  | 0.7  | 事故影响中等   |
| 12-24 个月 | 0.4  | 事故影响较小   |
| > 24 个月  | 0.2  | 仅作长期趋势参考 |

**时间衰减后事故数计算公式**：

```
n_decayed = Σ(各事故 × 对应时间区间衰减权重)
```

> 示例：某道路近 6 个月 2 起、6-12 个月 1 起、12-24 个月 1 起，则 n_decayed = 2×1.0 + 1×0.7 + 1×0.4 = 3.1。

统计窗口与衰减权重可在 F06 配置，默认采用上表。

#### 5.2.3 各指标标准化得分（0-100，越高风险越大）

风险子指标同样按输入类型分为固定分值和分段线性两类。

**A. 分类/二值指标：采用固定分值表**

| 指标     | 输入条件                    | 固定得分 | 业务解释       |
| ------ | ----------------------- | ---- | ---------- |
| 事故严重度  | 高（伤亡 ≥ 1 人或财产损失 ≥ 10 万） | 90   | 事故后果严重     |
|        | 中（轻微伤或财产损失 1-10 万）      | 60   |            |
|        | 低（无伤亡、财产损失 < 1 万）       | 20   |            |
| 交通事件频率 | 频发（月均 ≥ 3 次）            | 90   | 违章、拥堵等事件频繁 |
|        | 偶发（月均 1-2 次）            | 60   |            |
|        | 极少（月均 < 1 次）            | 20   |            |
| 流量修正系数 | 交通流量等级「高」               | 90   | 流量大，风险叠加   |
|        | 交通流量等级「中」               | 60   |            |
|        | 交通流量等级「低」               | 20   |            |
|        | 无数据                     | 60   | 数据缺失按中分处理  |

**B. 连续数值指标：采用分段线性函数**

| 指标   | 输入变量                 | 得分公式                                                                                                                                                                 | 说明              |
| ---- | -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------- |
| 事故频率 | 时间衰减后事故数 `n_decayed` | `n_decayed ≥ 5` 时：score = 90 + (n_decayed-5) × 2，上限 100<br>`2 ≤ n_decayed < 5` 时：score = 40 + (n_decayed-2) × 16.67<br>`n_decayed < 2` 时：score = 20 + n_decayed × 10 | 事故次数越多、越近期，风险越高 |

> 说明：固定分值表和线性函数的分段断点可在 F06 中配置，本方案先给出默认业务规则。

#### 5.2.4 风险指数

```
风险指数 = Σ（指标标准化得分 × 指标权重）
```

结果保留 1 位小数。

#### 5.2.5 风险等级阈值（新增，与复杂度模型对齐）

| 等级  | 阈值             |
| --- | -------------- |
| 低风险 | 风险指数 < 40      |
| 中风险 | 40 ≤ 风险指数 ≤ 69 |
| 高风险 | 风险指数 ≥ 70      |

> 地图可视化 F03 的风险专题图层颜色阈值，同步采用本表。

### 5.3 场景适配模型

#### 5.3.1 场景列表与门限条件（F06 可配置）

| 场景名称 | 最低车道数 | 允许机非混行 | 最低照明等级 | 允许最大坡度 |
| ---- | ----- | ------ | ------ | ------ |
| 城区道路 | 4     | 否      | 2      | 8%     |
| 园区道路 | 2     | 是      | 1      | 12%    |
| 环卫作业 | 2     | 是      | 1      | 10%    |
| 物流配送 | 2     | 否      | 2      | 8%     |
| 公交运营 | 4     | 否      | 3      | 6%     |

#### 5.3.2 连续加权打分规则

每个场景从 4 个维度打分，每个维度满分 25 分：

| 维度   | 得分规则                                                             |
| ---- | ---------------------------------------------------------------- |
| 车道数  | 实际车道数 ≥ 最低车道数：25 分；否则 0 分                                        |
| 机非混行 | 若道路存在机非混行，且场景「允许机非混行」为「是」：25 分；道路不存在机非混行：25 分；道路存在机非混行但场景不允许：0 分 |
| 照明等级 | 实际照明等级 ≥ 最低照明等级：25 分；否则 0 分                                      |
| 最大坡度 | 实际最大坡度 ≤ 允许最大坡度：25 分；否则 0 分                                      |

```
场景适配度得分 = Σ 各维度得分
```

结果范围为 0-100 分，保留 1 位小数。

#### 5.3.3 适配状态判定

| 状态  | 条件                      |
| --- | ----------------------- |
| 适用  | 场景适配度得分 ≥ 适配阈值（默认 60 分） |
| 不适用 | 场景适配度得分 < 适配阈值          |

#### 5.3.4 最佳适配场景

取所有场景中「场景适配度得分最高且状态为适用」的场景作为「最佳适配场景」。
若所有场景均不适用，则「最佳适配场景」显示为「无」，并在日志中标注。

## 6. 综合得分计算

### 6.1 公式

将复杂度指数、风险指数转换为「安全性/易行性」得分后，与场景适配度得分做加权平均：

```
综合得分 = (100 - 复杂度指数) × 复杂度权重
       + (100 - 风险指数) × 风险权重
       + 场景适配度得分 × 场景适配权重
```

其中：
- 复杂度权重、风险权重、场景适配权重取自 F06「融合模型」子模型权重配置；
- 三项权重之和必须等于 1；
- 结果截断到 [0, 100]，保留 1 位小数。

### 6.2 默认权重

| 子模型    | 默认权重 |
| ------ | ---- |
| 复杂度模型  | 0.33 |
| 风险模型   | 0.34 |
| 场景适配模型 | 0.33 |

### 6.3 示例

假设某道路：
- 复杂度指数 = 60 → 转换后 40
- 风险指数 = 50 → 转换后 50
- 场景适配度得分 = 80
- 默认权重 0.33 / 0.34 / 0.33

```
综合得分 = 40 × 0.33 + 50 × 0.34 + 80 × 0.33 = 56.2
```

按 6.4 映射规则，属于「限制开放」。

### 6.4 综合得分与开放等级映射

| 建议开放等级 | 综合得分阈值         |
| ------ | -------------- |
| 完全开放   | 综合得分 ≥ 85      |
| 条件开放   | 70 ≤ 综合得分 < 85 |
| 限制开放   | 50 ≤ 综合得分 < 70 |
| 禁止开放   | 综合得分 < 50      |

## 7. 融合规则与开放等级判定

### 7.1 判定顺序

采用「准入条件前置拦截、融合规则优先、综合得分映射兜底」的三段式流程：

1. **第一步：硬性准入条件检查**
   - 检查道路是否满足最低安全准入条件（见 7.2）；
   - 任一条件不满足，直接输出「建议开放等级 = 禁止开放」，并记录未满足条件；
   - 不再进入后续子模型计算与融合规则匹配。

2. **第二步：子模型计算**
   - 计算复杂度指数/等级、风险指数/等级、各场景适配度得分/状态。

3. **第三步：融合规则匹配**
   - 按优先级从高到低依次匹配 F06 中启用的融合规则；
   - 融合规则条件既可以是子模型输出（如风险等级 = 高），也可以是外部数据/事件/状态（如道路类型、施工状态、死亡事故、气象预警）；
   - 首次命中即输出「建议开放等级」，不再继续匹配；
   - 全局禁止条件「风险等级 = 高 → 禁止开放」作为最高优先级隐式规则。

> 设计意图：综合得分映射解决「常规、可量化、连续差异」的排序问题；融合规则解决「政策红线、特殊区域、动态事件、一票否决」等无法或不应由公式表达的业务规则。

4. **第四步：综合得分映射兜底**
   - 若未命中任何融合规则，则使用 6.4 的映射表由综合得分确定「建议开放等级」。

5. **第五步：业务规则映射**
   - 根据确定的「建议开放等级」，从映射关系表（见 8）中获取适用自动驾驶等级、允许测试类型、限制时段、管控说明。

### 7.2 硬性准入条件

以下任一条件不满足，该道路直接判定为「禁止开放」：

| 准入条件       | 阈值      | 数据来源             | 缺失处理                |
| ---------- | ------- | ---------------- | ------------------- |
| 最小车道宽度     | ≥ 3.0 m | F01 道路建档（建议新增字段） | 数据缺失时，该条件视为「不满足」    |
| 最大坡度       | ≤ 6%    | F01 道路建档补充数据     | 数据缺失时，该条件视为「不满足」    |
| 交通信号覆盖率    | ≥ 80%   | 复杂度模型「信号灯完整度」    | 覆盖率数据缺失时，该条件视为「不满足」 |
| 视距严重不足路段占比 | < 10%   | F01 道路建档（建议新增字段） | 数据缺失时，该条件视为「不满足」    |

> 说明：
> 1. 准入条件阈值可在 F06 配置；
> 2. 准入检查失败时，评估日志需明确记录哪一项条件未满足及实际值；
> 3. 本期如部分字段未采集，可先放开该条件或提高阈值，但需在配置版本中留痕。

### 7.3 默认融合规则（与 F06 保持一致）

融合规则的核心价值在于处理**综合得分公式无法表达或不应由公式表达**的外部条件、政策法规、特殊事件和动态状态。以下默认规则按优先级从高到低排列：

| 优先级 | 条件描述                    | 动作     | 规则来源    | 默认状态 |
| --- | ----------------------- | ------ | ------- | ---- |
| 1   | 道路类型 = 高速公路/城市快速路       | 禁止开放   | 政策法规    | 启用   |
| 2   | 近 3 个月发生死亡事故            | 禁止开放   | 一票否决    | 启用   |
| 3   | 道路位于学校/医院/政府周边 200 米范围内 | 禁止开放   | 特殊区域保护  | 启用   |
| 4   | 道路状态 = 施工中/封路           | 禁止开放   | 动态状态    | 启用   |
| 5   | 气象预警等级 = 红色             | 临时禁止开放 | 外部预警    | 启用   |
| 6   | 风险等级 = 高                | 禁止开放   | 安全红线    | 启用   |
| 7   | 复杂度 = 高 AND 风险 = 高      | 限制开放   | 子模型组合策略 | 启用   |
| 8   | 风险 = 中 AND 场景适配 < 60    | 限制开放   | 子模型组合策略 | 禁用   |

> 说明：
> 1. 优先级 1-5 为外部/非量化条件，无法通过调整综合得分公式实现，体现融合规则不可替代的价值；
> 2. 优先级 6-8 为子模型输出组合规则，业务上可通过综合得分映射近似表达，但保留为显式规则可提高可解释性和安全可控性；
> 3. 全局禁止条件「风险等级 = 高 → 禁止开放」作为最高优先级隐式规则与规则 6 重复，保留其一即可。


### 7.4 规则命中记录

评估日志中需记录：
- 是否命中融合规则；
- 命中规则的优先级、条件描述、动作、规则来源（政策法规/特殊区域/动态状态/外部预警/子模型组合）；
- 若命中外部条件规则，记录触发该规则的具体数据值（如道路类型、事故编号、气象预警等级、施工状态）；
- 若未命中，记录「未命中融合规则，采用综合得分映射」。

## 8. 开放等级业务规则映射

采用场景类型（载人/载货/环卫/物流）作为「允许测试类型」口径。

### 8.1 开放等级映射表

| 建议开放等级 | 适用自动驾驶等级     | 允许测试类型      | 限制时段        | 管控要求      |
| ------ | ------------ | ----------- | ----------- | --------- |
| 完全开放   | L2 / L3 / L4 | 载人、载货、环卫、物流 | 无限制         | 无特殊要求     |
| 条件开放   | L3 / L4      | 载货、环卫、物流    | 06:00-22:00 | 需额外提交安全评估 |
| 限制开放   | L4           | 环卫、物流       | 09:00-17:00 | 需安全员全程监控  |
| 禁止开放   | —            | —           | —           | 禁止任何测试活动  |

> 说明：「公交运营」场景在映射表中未列出，因其对道路条件要求更高，建议单独扩展一条映射规则，本期可先按「限制开放」处理，并在 PRD 中明确。

### 8.2 限制条件自动生成规则

在基础映射表之上，系统根据评估结果自动叠加限制条件，形成最终「管控要求」文本。限制条件由模板库 + 触发条件组合生成，支持在 F06 配置扩展。

| 触发条件                   | 自动限制条件                  | 优先级 |
| ---------------------- | ----------------------- | --- |
| 复杂度指数 ≥ 70（高复杂度）       | 最高限速 40 km/h            | 中   |
| 风险指数 ≥ 70（高风险）         | 仅限日间开放（06:00-20:00）     | 高   |
| 最大坡度 > 6%              | 需安全员随车                  | 中   |
| 场景适配度得分 < 60           | 禁止无人化测试，需安全员全程监控        | 高   |
| 机非混行 = 是 AND 场景不允许机非混行 | 限制测试时段为平峰期（10:00-16:00） | 中   |

**生成规则**：
1. 先取 8.1 中对应开放等级的「管控要求」作为基础文本；
2. 遍历上表触发条件，命中则追加对应限制条件；
3. 多条限制条件以「；」连接，按优先级高→低排序；
4. 最终管控要求写入评估结果表 `control_requirements` 字段，并在判定依据中说明来源。

> 示例：某道路判定为「限制开放」，同时命中「高风险」和「最大坡度 > 6%」，则最终管控要求为：
> 「需安全员全程监控；仅限日间开放（06:00-20:00）；需安全员随车。」

## 9. 评估触发与版本管理

### 9.1 触发方式

| 触发类型   | 触发条件               | 评估范围         |
| ------ | ------------------ | ------------ |
| 定期评估   | 每日凌晨 2:00 系统自动执行   | 全量道路（含已关闭道路） |
| 道路信息变更 | 道路建档/编辑保存后         | 单条道路         |
| 道路状态变更 | 道路状态从「暂停」恢复为「开放」时  | 单条道路         |
| 手动触发   | 用户在 F05 点击「手动触发评估」 | 全量道路（含已关闭道路） |
| 重新评估   | 用户在 F05 点击「重新评估」   | 单条道路         |

### 9.2 已关闭道路参与评估

已关闭道路参与所有全量/单条评估，评估结果正常记录。但已关闭道路不在测试示范准入申请的可选道路列表中展示（状态过滤逻辑不变）。

### 9.3 配置版本绑定

- 每次保存 F06 评估指标配置，自动生成新版本号（如 V3.3）；
- 每次评估执行时，记录当前生效的配置版本号；
- 评估结果表字段 `config_version` 必填；
- 评估日志中保存「配置快照」：复杂度模型权重、风险模型权重、场景适配模型权重、硬性准入条件阈值、融合规则列表、映射关系表、限制条件模板库、各阈值。

### 9.4 历史版本对比

F05「评估记录对比」选择两个版本后，展示：
- 综合得分变化；
- 复杂度指数变化；
- 风险指数变化；
- 场景适配得分变化；
- 开放等级变化；
- 命中融合规则变化（新增/消失/动作变化）。

### 9.5 评估结果输出格式

每条道路的评估结果除基础字段外，还需输出以下结构化数据，供 F05 详情页、F07 日志详情页及地图可视化使用。

### 9.6 判定依据说明生成

评估结果需自动生成结构化判定依据，用于详情页展示、日志留痕及对外报告导出。判定依据包含以下要素：

| 要素       | 内容                                                                           |
| -------- | ---------------------------------------------------------------------------- |
| 准入条件检查结果 | 是否通过，未通过项及实际值                                                                |
| 复杂度评分及等级 | 复杂度指数、等级、各子指标贡献                                                              |
| 风险评分及等级  | 风险指数、等级、事故时间衰减说明、各子指标贡献                                                      |
| 场景适配排序   | 各场景适配度得分及状态，最佳适配场景                                                           |
| 命中规则     | 融合的优先级、条件、动作、规则来源（政策法规/特殊区域/动态状态/外部预警/子模型组合）；若命中外部条件，记录触发数据值；若未命中则说明采用综合得分映射 |
| 综合得分     | 计算公式、各子模型转换后得分及权重                                                            |
| 映射结果     | 开放等级、适用自动驾驶等级、允许测试类型、限制时段                                                    |
| 限制条件来源   | 自动触发的限制条件及触发原因                                                               |
| 数据缺失说明   | 哪些指标采用默认值，默认原因                                                               |

**自然语言示例**：

> 该道路复杂度指数 60.0（中复杂度），风险指数 45.0（中风险），最佳适配场景为「物流配送」（适配度 80.0）。未命中融合规则，按综合得分 56.2 映射为「限制开放」。因风险等级为中且存在陡坡路段，自动附加限制条件「需安全员全程监控；需安全员随车」。部分字段（视距不足路段占比）缺失，按默认中分处理。

> 该道路复杂度指数 35.0（低复杂度），风险指数 40.0（中风险），场景适配度 85.0。但道路类型为「城市快速路」，命中融合规则优先级 1（政策法规：道路类型 = 高速公路/城市快速路 → 禁止开放），因此直接判定为「禁止开放」，不再进入综合得分映射。

## 14. 二期扩展建议（参考原稿）

本期方案优先保证与现有 PRD 页面口径一致、可快速落地。待数据条件成熟后，可从原稿中吸收以下高级能力作为二期增强：
