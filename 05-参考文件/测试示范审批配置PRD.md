## 2 名词解释

| 名词/缩写  | 解释                                                           |
| ------ | ------------------------------------------------------------ |
| 测试示范主体 | 发起准入申请的企业或机构类型，如整车制造企业、系统运营商、零部件制造商、互联网服务商、科研院所/高校、其他        |
| 测试示范对象 | 参与测试示范活动的车辆对象类型，如乘用车、商用车、专业作业车、功能型低速无人车、其他                   |
| 测试示范类型 | 准入申请流程类型，按业务大类与申请子类型组合形成，如道路测试初次申请、示范应用变更申请、商业化试点新增相同配置车辆申请等 |
| 适用主体   | 某一测试示范类型允许发起申请的主体类型集合                                        |
| 适用对象   | 某一测试示范类型允许纳入申请的车辆对象类型集合                                      |
| 配置快照   | 申请提交时固化的主体、对象、类型、材料目录、审批流程等配置版本，用于保证历史申请不受后续配置变更影响           |
| 材料项库   | 可被流程材料目录引用的基础材料项集合，维护材料代码、名称、提供方、格式要求、状态等基础信息                |
| 流程材料目录 | 针对某一测试示范类型配置的材料清单，包含材料项、是否必填、排序、模板要求、填报说明等流程级属性              |
| 目录版本   | 流程材料目录发布后形成的版本记录，新发起申请使用当前已发布版本，历史申请保留提交时目录快照                |

## 3 业务背景与使用角色
### 3.1 业务背景
- 模块定位：测试示范审批配置是准入信息管理与准入申请流程之间的配置中心，负责维护可申请主体、可申请车辆对象、可发起申请流程类型及其关联规则。
- 核心特点：三类配置互相关联，主体配置被企业登记引用，对象配置被车辆登记引用，类型配置被准入申请引用，并进一步关联申请材料目录与审批流程。
- 关联模块关系：企业信息登记引用测试示范主体配置，车辆信息登记引用测试示范对象配置，准入申请引用测试示范类型配置，审批管理按类型配置关联的流程展示审批节点。
- 建设必要性：避免主体类型、车辆类型、申请流程类型分散写死在各页面中，支持后续新增测试示范对象或申请流程时，通过配置完成页面展示与流程准入控制。
### 3.2 使用角色

| 角色 | 该模块中的职责                             | 客户端 |
| --------- | ----------------------------------- | --- |
| 平台管理员 | 维护主体配置、对象配置、类型配置、材料项库、流程材料目录和目录版本，执行新增、编辑、启停、删除、发布等配置操作 | 管理端 |
| 政府监管人员 | 查看测试示范配置、核对准入申请规则与监管口径是否一致          | 管理端 |
| 第三方专业管理机构 | 查看配置后的申请类型、材料目录、审批流程，用于受理与初审业务理解    | 管理端 |
| 企业用户 | 不直接维护配置，在企业登记、车辆登记、准入申请页面使用配置项      | 企业端 |

### 3.3 核心目标与价值

| 目标维度  | 说明                                    |
| ----- | ------------------------------------- |
| 统一字典  | 将申请主体类型、车辆对象类型、申请流程类型统一维护，避免各页面枚举不一致  |
| 配置联动  | 主体配置联动企业登记，对象配置联动车辆登记，类型配置联动准入申请      |
| 流程扩展  | 支持新增测试示范类型后，关联适用主体、适用对象、材料目录、审批流程     |
| 材料复用  | 通过材料项库统一维护可复用材料，再按测试示范类型组合生成流程材料目录     |
| 版本追溯  | 目录发布形成版本记录，支持版本查看、对比和历史申请快照保护              |
| 评审可视化 | 页面可清晰展示某一申请流程类型的主体范围、对象范围、材料目录与审批流程关系 |

## 5 功能详细设计
### 5.1 F01-F03 测试示范审批配置页面
#### 5.1.1 页面布局

页面自上而下分为 5 个区域：

1. **页面标题区**：标题为“测试示范审批配置”
2. **配置说明区**：以轻量提示条展示三类配置关系：主体配置关联企业类型、对象配置关联车辆类型、类型配置关联申请流程类型。
3. **Tab 页签区**：包含“测试示范主体配置”“测试示范对象配置”“测试示范类型配置”三个 Tab。
4. **查询与列表区**：每个 Tab 内展示对应查询条件、数据列表、分页区，分页见附录 A.1。
5. **操作弹窗区**：新增、编辑以表单弹窗展示，启用/停用、删除以二次确认弹窗展示，二次确认见附录 A.6。

7. 用户切换到“目录版本管理”Tab → 左侧选择测试示范类型，右侧展示该类型目录版本列表；用户可查看版本详情、选择两个版本进行对比、将历史版本复制为新草稿。

### 6.3 配置生效规则

| 场景      | 生效规则                                                            |
| ------- | --------------------------------------------------------------- |
| 新增配置    | 保存成功后立即可在对应配置列表中展示；启用状态下可被新增业务引用                                |
| 编辑配置    | 保存成功后生成新配置版本；未提交的新增业务使用最新配置                                     |
| 历史申请    | 历史申请保留提交时配置快照，不随配置变更自动调整                                        |
| 主体/对象状态 | 主体配置和对象配置不再设置单独的登记选择开关；状态为已启用时可进入登记、类型配置与申请筛选可选项，状态为已停用时仅保留历史展示 |
| 停用配置    | 停用后未来新增业务和字段变更不再可选；历史企业、车辆、申请记录继续展示提交或保存时的配置名称                  |
| 删除配置    | 删除采用逻辑删除；删除后未来新增业务和字段变更不再可选，历史企业、车辆、申请记录继续按快照展示                 |

| 车辆详情 | 车辆类型 | 配置快照 | 展示车辆保存时的对象名称，若已停用或已删除则展示对应标签 |

| 申请环节 | 联动规则 |

| 材料清单展示 | 按测试示范类型关联的材料目录版本加载材料清单，提交时固化材料目录快照 |
| 流程预览展示 | 按测试示范类型关联的审批流程版本展示流程节点，提交时固化审批流程快照 |

### 6.6 材料目录配置规则

| 规则项          | 规则说明                                                         |

| 一类流程一个当前发布目录 | 每个测试示范类型最多存在一个当前已发布材料目录版本，可存在一个草稿版本和多个历史版本                   |

| 发布影响边界       | 发布新目录版本后，仅新发起申请使用新版本；已提交申请继续使用提交时固化的目录快照                     |

| 删除材料项处理      | 删除采用逻辑删除；删除后不可再被新增目录选择，历史目录版本和历史申请快照不回写                      |

- [ ] 类型配置保存后，准入申请页面可按类型配置展示申请入口、筛选企业、筛选车辆、加载材料目录和流程预览。
- [ ] 停用或删除测试示范类型后，企业端不再展示该申请入口，历史申请不受影响。
### 10.4 联动规则验收
- [ ] 企业信息登记中的申请主体类型来源于已启用测试示范主体配置。
- [ ] 车辆信息登记中的车辆类型来源于已启用测试示范对象配置。
- [ ] 准入申请选择测试示范类型后，仅展示适用主体范围内的企业。
- [ ] 准入申请选择测试示范类型和申请主体后，仅展示适用对象范围内且属于该企业的车辆。
- [ ] 申请提交时固化配置快照，历史申请不随配置变更自动修改。
### 10.5 材料项库验收
- [ ] 页面存在“材料项库”Tab，列表字段包含材料代码、材料名称、提供方、格式要求、状态、更新时间、操作。
- [ ] 材料项库表单不包含“是否必填”字段。
- [ ] 支持新增、编辑、启用、停用、删除材料项。
- [ ] 停用或删除材料项后，流程材料目录新增选择中不再展示该材料项，历史目录版本和历史申请快照保留。
### 10.6 流程材料目录配置验收
- [ ] 页面存在“流程材料目录配置”Tab，左侧可选择测试示范类型，右侧展示该类型材料目录。
- [ ] 支持从材料项库添加已启用材料项到当前流程目录。
- [ ] 每个目录材料项可配置是否必填、排序号、上传格式、是否需要模板、模板文件、填报说明、适用条件。
- [ ] 支持保存草稿，草稿不影响企业端新发起申请展示的当前已发布目录。
- [ ] 支持发布目录版本，发布前需填写版本号、变更说明并二次确认。
### 10.7 目录版本管理验收
- [ ] 页面存在“目录版本管理”Tab，可按测试示范类型查看草稿、已发布、历史版本。
- [ ] 每个测试示范类型最多存在一个当前已发布目录版本。
- [ ] 支持查看版本详情，展示该版本的材料项、必填规则、排序、模板、填报说明和适用条件。
- [ ] 支持选择两个目录版本进行对比，展示新增、移除和属性变更材料项。
- [ ] 新发起申请使用当前已发布目录版本，历史申请继续使用提交时固化的目录快照。

### 11.2 A.2 删除确认组件
1. 用户点击“删除” → 弹出二次确认弹窗。
2. 确认弹窗内容：“确认删除该配置项吗？删除后新增业务将无法引用该配置，历史记录不受影响。”
3. 点击“确定” → 执行逻辑删除 + 记录删除操作日志 + 列表刷新。
4. 点击“取消” → 关闭确认弹窗，不执行删除。
### 11.3 A.3 导出组件

| 触发条件 | 行为 |
| -------- | ---- |
| 用户选中了特定数据 | 导出用户选择的数据，不支持跨页导出 |
| 用户未选择特定数据 | 弹出提示“是否导出所有数据？”，用户选择“是”则导出当前筛选条件下的所有数据，选择“否”则不执行操作 |

**导出格式：** .xlsx

**文件命名规则：** 测试示范审批配置 _YYYYMMDD.xlsx

### 11.4 A.4 附件上传组件

| 规则项 | 图片类 | 文档类 |
| ------ | ------ | ------ |
| 支持格式 | jpg/png | pdf/doc/docx |
| 单文件大小限制 | ≤5MB | ≤20MB |
| 多文件上传 | 支持 | 支持 |
| 预览/下载 | 查看态支持点击预览/下载 | 查看态支持点击下载 |

### 11.5 A.5 复选框组件
- 表头存在全选复选框时，勾选后选中当前页所有行。
- 支持单行勾选。
- 勾选状态用于导出、批量操作等场景。
- 全选仅作用于当前页数据，不支持跨页全选。
