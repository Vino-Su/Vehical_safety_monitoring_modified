## 2. 背景与问题

监控一张图是监管人员开展智能网联车辆实时监管的核心入口。该页面一方面需要关注车辆的实时运行状态，另一方面需要关注来自车辆、终端、围栏、时段合规、安全员状态等维度的预警信息。

现有页面已覆盖地图、车辆点位、预警标识、图层控制、车辆列表、轨迹和视频等基础能力，但整体呈现更接近普通后台页面，存在以下产品体验问题：

1. 地图作为核心监管画布的视觉权重不足，容易被顶部统计卡片和右侧固定列表挤压。
2. 车辆列表长期占用右侧区域，导致预警处置入口不够突出。
3. 顶部统计卡片展示方式偏普通，缺少实时态势和监管优先级。
4. 车辆实时状态与预警信息之间的联动关系不够清晰。
5. 图层控制与业务筛选边界容易混淆，后续扩展后可能导致操作复杂。

本 PRD 旨在将页面升级为“车辆实时状态 + 预警处置”双主线融合的监管态势驾驶舱。

## 3. 产品定位

### 3.1 页面定位

监控一张图是以地图为核心的实时监管页面，用于集中展示智能网联车辆运行状态、预警事件、事故点位、开放道路、电子围栏和设施设备，并支持监管人员完成定位、研判、轨迹回放、视频核验和处置跳转。

### 3.2 核心用户

| 用户角色 | 使用目标 |
| --- | --- |
| 监管部门人员 | 实时掌握车辆运行态势，发现异常并快速定位处置 |
| 第三方专业管理机构 | 辅助开展车辆运行监测、预警核验和事故跟踪 |
| 平台运维人员 | 查看车辆数据上报状态、终端在线状态和地图图层状态 |

### 3.3 核心目标

1. 让用户一眼看到当前有多少车辆在运行、哪些车辆异常、异常在哪里。
2. 支持从预警事件快速定位到车辆、轨迹、视频和处置入口。
3. 支持从车辆对象快速查看实时状态、当前预警和历史轨迹。
4. 保持地图主画布的主导地位，避免被列表和卡片弱化。
5. 明确图层与筛选边界，降低复杂监管场景下的操作混乱。

## 4. 设计原则

### 4.1 双主线融合

页面需同时服务两类监管任务：

1. 车辆实时状态监管：车辆在哪里、是否在线、驾驶模式、速度、是否数据延迟。
2. 车辆预警信息监管：哪些车辆有预警、预警等级、预警类型、是否已处理、如何处置。

两条主线不应割裂展示，而应通过地图、右侧预警面板、车辆详情卡和底部车辆列表形成联动。

### 4.2 地图优先

地图是监控一张图的第一视觉中心。页面布局需保证地图拥有最大可视面积，统计、筛选、图层、列表均应作为地图监管能力的辅助组件。

### 4.3 预警优先

预警事件是监管人员最需要立即响应的信息。右侧面板默认展示预警队列，严重预警置顶，支持一键定位和进入研判。

### 4.4 最小充分信息

右侧区域空间有限，不承载完整详情。其职责是展示当前最需要处理的预警，以及当前选中车辆的最小充分上下文。完整详情通过弹窗、抽屉或二级页面承载。

### 4.5 图层与筛选分离

图层回答“地图上显示哪些类型的对象”，筛选回答“这些对象中展示哪些数据”。二者需在交互入口和信息结构上明确区分。

## 5. 页面整体布局

### 5.1 推荐布局

页面采用“顶部实时态势条 + 地图主画布 + 右侧预警上下文面板 + 底部车辆列表抽屉”的结构。

```text
顶部实时态势条
在线车辆 | 自动驾驶 | 严重预警 | 一般预警 | 当日事故 | 数据刷新状态

主体区域
左侧/中间：地图主画布
  - 车辆点位
  - 预警标识
  - 事故点位
  - 电子围栏
  - 开放道路
  - 轨迹
  - 地图工具栏
  - 图例

右侧：预警与车辆上下文面板
  - 默认展示全局预警队列
  - 选中车辆后展示车辆上下文卡
  - 支持该车预警/全局预警切换

底部：车辆状态列表抽屉
  - 默认收起
  - 展开后查看全量车辆状态列表
```

### 5.2 区域尺寸建议

| 区域 | 展示方式 | 尺寸建议 |
| --- | --- | --- |
| 顶部实时态势条 | 横向紧凑状态条 | 高度 48px-64px |
| 地图主画布 | 主体核心区域 | 占主体宽度 70%-80% |
| 右侧预警面板 | 固定宽度侧栏 | 360px-420px |
| 底部车辆列表抽屉 | 可展开/收起 | 收起 40px-48px，展开不超过主体高度 40% |
| 图层/筛选浮层 | 地图内浮层 | 宽度 280px-320px |

### 5.3 视觉规范

页面需遵循 Ant Design 企业后台风格：

1. 主色使用 `#1677ff`。
2. 背景使用 `#f0f2f5`，地图容器和面板保持浅色企业风格。
3. 文本颜色使用 `#000000d9`、`#00000073`、`#00000040`。
4. 预警等级色建议：严重 `#ff4d4f`，一般 `#faad14`，正常 `#52c41a`。
5. 图标使用统一线性图标，不使用 emoji 作为正式页面图标。
6. 面板圆角、阴影、间距需与现有 Ant Design 风格一致。

## 6. 顶部实时态势条

### 6.1 功能定位

顶部实时态势条用于快速展示当前监管态势，不再采用厚重的 4 列统计卡片。其核心价值是帮助监管人员快速判断当前是否存在风险。

### 6.2 指标设计

| 指标     | 展示内容                | 交互                 |
| ------ | ------------------- | ------------------ |
| 在线车辆   | 在线数 / 总数、在线率        | 点击后筛选在线车辆          |
| 自动驾驶   | 自动驾驶模式车辆数           | 点击后筛选自动驾驶车辆        |
| 严重预警   | 严重预警数量              | 点击后筛选严重预警，右侧队列同步切换 |
| 一般预警   | 一般预警数量              | 点击后筛选一般预警          |
| 当日事故   | 当日事故数量              | 点击后开启事故点位图层        |
| 数据刷新状态 | 实时同步中、最近刷新时间、数据延迟状态 | 鼠标悬停展示刷新说明         |

### 6.3 展示要求

1. 严重预警和当日事故应具备更高视觉优先级。
2. 在线车辆、自动驾驶车辆为态势背景信息，不应与严重预警同等强调。
3. 数据刷新状态需明确显示最近同步时间。
4. 点击态势指标后，地图、右侧预警队列、底部车辆列表需同步联动。

## 7. 地图主画布

### 7.1 地图定位

地图主画布是页面第一主角，用于展示车辆、预警、事故、围栏、道路和轨迹等空间信息。

### 7.2 车辆点位

车辆 marker 需同时表达车辆运行状态与预警状态。

| 状态     | 展示规则            |
| ------ | --------------- |
| 自动驾驶模式 | 蓝色车辆点位          |
| 人工驾驶模式 | 绿色车辆点位          |
| 测试模式   | 紫色车辆点位          |
| 离线     | 灰色车辆点位          |
| 存在预警   | 车辆点位外层增加红/橙预警光圈 |
| 多条预警   | 点位右上角显示预警数量角标   |

### 7.3 预警表达

预警不应完全替代车辆状态，而应叠加在车辆点位上：

1. 车辆本体颜色表达驾驶模式或在线状态。
2. 外层光圈表达预警等级。
3. 角标表达当前预警数量。
4. 严重预警车辆应保持可见，不被聚类淹没。

### 7.4 地图气泡

点击车辆 marker 后展示轻量信息气泡，内容控制在一屏内：

| 信息类型 | 字段 |
| --- | --- |
| 基础信息 | 车牌号、VIN、所属企业 |
| 实时状态 | 驾驶模式、速度、在线状态 |
| 预警信息 | 当前预警数量、最高预警等级 |
| 快捷操作 | 查看轨迹、实时视频、查看详情 |

完整字段不在地图气泡中全部展开，避免遮挡地图。

### 7.5 聚类要求

1. 地图缩放级别较低时，普通车辆进入聚类展示。
2. 严重预警车辆不进入普通聚类，保持独立可见。
3. 聚类点点击后应放大地图并展开该区域车辆。
4. 聚类点需展示区域内车辆数量和预警数量。

## 8. 右侧预警与车辆上下文面板

### 8.1 面板定位

右侧面板宽度有限，不承载完整详情。其职责是：

1. 默认展示当前最重要的预警队列。
2. 帮助用户从预警快速定位车辆。
3. 选中车辆后展示车辆最小充分上下文。
4. 提供轨迹、视频、详情、处置等快捷入口。

### 8.2 未选中车辆状态

页面初始进入或未选中车辆时，右侧面板展示全局预警队列。

```text
右侧面板
├─ 预警概览：严重 2 | 一般 5 | 待处理 4
├─ 快速筛选：全部 | 严重 | 一般 | 未处理
├─ 预警队列
└─ 底部入口：车辆搜索 / 展开车辆列表
```

当预警队列为空时，展示空状态：

```text
暂无待处理预警
点击地图车辆可查看车辆实时状态、轨迹和视频。
```

### 8.3 选中车辆状态

用户通过点击地图车辆、预警卡片或车辆列表行选中车辆后，右侧面板顶部出现“选中车辆上下文卡”。

```text
鄂F·D010    告警中
自动驾驶 L4 | 15km/h | 14:32:20 上报
当前预警 2 条

[轨迹] [视频] [详情]
```

选中车辆后，下方预警队列默认切换为“该车预警”，并提供“全局预警”切换入口。

### 8.4 车辆上下文卡字段

| 字段     | 说明             |
| ------ | -------------- |
| 车牌号    | 主标题            |
| 当前状态   | 在线、离线、告警中、数据延迟 |
| 驾驶模式   | 自动驾驶、人工驾驶、测试模式 |
| 自动驾驶等级 | L2/L3/L4 等     |
| 实时速度   | km/h           |
| 最后上报时间 | 精确到秒           |
| 当前预警数量 | 严重/一般分级        |

### 8.5 预警队列卡片

预警队列卡片采用“列表态 + 展开态 + 详情态”三层信息结构。

#### 8.5.1 列表态

列表态只展示最关键的 5 个信息：

```text
[严重] 电子围栏越界              14:10
鄂F·J056 | B公司
襄阳高新区测试道路 · 2.3km
```

字段要求：

| 字段 | 说明 |
| --- | --- |
| 预警等级 | 严重/一般 |
| 预警类型 | 如电子围栏越界、终端异常、车辆运行异常 |
| 发生时间 | 时分展示，详情中展示完整时间 |
| 车辆号牌 | 对应车辆 |
| 企业 | 所属企业 |
| 位置摘要 | 道路或区域名称 |

#### 8.5.2 展开态

点击预警卡片后在队列内展开，展示：

| 字段 | 说明 |
| --- | --- |
| 预警编号 | 唯一编号 |
| VIN | 对应车辆 VIN |
| 处理状态 | 未处理、处理中、已处理 |
| 触发规则 | 预警触发说明 |
| 操作 | 定位、看轨迹、实时视频、进入处置 |

#### 8.5.3 详情态

点击“详情”或“进入处置”后打开详情抽屉或跳转处置页面，展示完整预警内容、处理记录、关联轨迹、视频截图和处置结果。

### 8.6 右侧面板容量要求

1. 右侧面板宽度为 360px-420px 时，一屏应至少可展示 5 条列表态预警。
2. 单条列表态预警高度建议控制在 72px-88px。
3. 展开态一次只建议展开一条预警，展开其他预警时自动收起上一条。
4. 文本超出需省略，鼠标悬停展示完整内容。

## 9. 底部车辆状态列表抽屉

### 9.1 功能定位

车辆列表用于批量查看和筛选车辆，不应默认长期占用右侧主面板。建议以底部抽屉形式展示。

### 9.2 收起态

收起态展示车辆概览：

```text
车辆状态列表  在线 186 / 离线 3 / 预警 7    展开
```

### 9.3 展开态

展开后展示完整车辆列表，字段包括：

| 字段 | 说明 |
| --- | --- |
| 车牌号 | 支持点击定位 |
| VIN | 车辆唯一识别码 |
| 企业 | 所属企业 |
| 申请类型 | 道路测试、示范应用、商业化试点 |
| 驾驶模式 | 自动驾驶、人工驾驶、测试模式 |
| 速度 | 当前速度 |
| 在线状态 | 在线、离线、数据延迟 |
| 预警数量 | 当前预警数量 |
| 最后上报时间 | 最近数据上报时间 |
| 操作 | 定位、轨迹、视频、详情 |

### 9.4 联动要求

1. 点击列表行后，地图自动定位并高亮车辆。
2. 右侧面板顶部展示选中车辆上下文卡。
3. 如该车存在预警，右侧队列默认切换至该车预警。
4. 列表筛选结果同步影响地图车辆点位。

## 10. 图层与筛选设计

### 10.1 边界定义

| 类型 | 回答的问题 | 示例 |
| --- | --- | --- |
| 图层控制 | 地图上显示哪些类型的对象 | 是否显示车辆、预警、事故、围栏 |
| 数据筛选 | 某类对象中显示哪些数据 | 只看严重预警、只看自动驾驶车辆 |

### 10.2 图层控制

图层控制只管理地图对象显隐，不承载复杂业务筛选。

| 分组 | 图层项 | 默认状态 |
| --- | --- | --- |
| 底图 | 电子地图 | 开 |
| 底图 | 卫星影像 | 关 |
| 业务图层 | 车辆实时位置 | 开 |
| 业务图层 | 预警标识 | 开 |
| 业务图层 | 事故点位 | 关 |
| 业务图层 | 行驶轨迹 | 关 |
| 业务图层 | 电子围栏 | 关 |
| 业务图层 | 开放道路 | 开 |
| 业务图层 | 设施设备 | 关 |
| 辅助显示 | 图例 | 开 |
| 辅助显示 | 聚合点 | 开 |
| 辅助显示 | 车辆标签 | 关 |

### 10.3 数据筛选

数据筛选按业务对象分别组织，不放入图层面板。

#### 10.3.1 车辆筛选

| 筛选项 | 说明 |
| --- | --- |
| 车牌号/VIN | 支持模糊搜索 |
| 企业 | 下拉选择 |
| 申请类型 | 道路测试、示范应用、商业化试点 |
| 驾驶模式 | 自动驾驶、人工驾驶、测试模式 |
| 在线状态 | 在线、离线、数据延迟 |
| 是否存在预警 | 全部、有预警、无预警 |

#### 10.3.2 预警筛选

| 筛选项 | 说明 |
| --- | --- |
| 预警等级 | 严重、一般 |
| 预警类型 | 车辆运行异常、碰撞、终端异常、临牌预警、电子围栏越界、时段合规、安全员状态 |
| 处理状态 | 未处理、处理中、已处理 |
| 发生时间 | 今日、近 7 日、自定义 |
| 企业 | 下拉选择 |

#### 10.3.3 事故筛选

| 筛选项 | 说明 |
| --- | --- |
| 事故等级 | 无伤亡、轻伤、重伤等 |
| 事故类型 | 碰撞、刮擦、追尾等 |
| 是否历史事故 | 当日事故、历史事故 |
| 发生时间 | 今日、近 7 日、自定义 |

### 10.4 典型边界示例

| 用户操作 | 归属 |
| --- | --- |
| 显示/隐藏预警标识 | 图层控制 |
| 只看严重预警 | 预警筛选 |
| 显示/隐藏车辆实时位置 | 图层控制 |
| 只看自动驾驶车辆 | 车辆筛选 |
| 显示/隐藏事故点位 | 图层控制 |
| 只看历史事故 | 事故筛选 |

## 11. 轨迹与视频

### 11.1 轨迹展示

触发入口：

1. 地图车辆气泡中的“查看轨迹”。
2. 右侧车辆上下文卡中的“轨迹”。
3. 预警卡片展开态中的“看轨迹”。
4. 底部车辆列表操作列中的“轨迹”。

展示要求：

1. 地图叠加当天轨迹路径。
2. 底部浮出轨迹控制面板。
3. 支持开始、暂停、快进、快退、倍速、拖动进度条。
4. 轨迹控制面板需展示当前时间、速度、驾驶模式。
5. 轨迹播放时右侧面板仍保留选中车辆上下文。

### 11.2 视频展示

触发入口：

1. 地图车辆气泡中的“实时视频”。
2. 右侧车辆上下文卡中的“视频”。
3. 预警卡片展开态中的“实时视频”。
4. 底部车辆列表操作列中的“视频”。

展示要求：

1. 以浮窗或右侧扩展面板展示视频。
2. 支持多个摄像头缩略图切换。
3. 单个摄像头画面支持截图、本地录像、全屏。
4. 车辆离线或摄像头不可用时展示“视频流不可用”。
5. 支持切换历史回放，选择日期时间范围并查询录像片段。

## 12. 关键交互流程

### 12.1 从预警定位车辆

1. 用户进入页面，右侧默认展示全局预警队列。
2. 用户点击严重预警卡片。
3. 地图自动定位到对应车辆。
4. 对应车辆 marker 高亮，打开轻量气泡。
5. 右侧顶部出现选中车辆上下文卡。
6. 右侧队列切换为该车预警。
7. 用户可继续查看轨迹、视频或进入处置。

### 12.2 从车辆查看预警

1. 用户在地图上点击车辆 marker。
2. 地图展示车辆轻量气泡。
3. 右侧出现车辆上下文卡。
4. 如车辆存在预警，右侧展示该车预警列表。
5. 用户点击预警后展开预警详情。

### 12.3 从顶部指标筛选风险

1. 用户点击顶部“严重预警”。
2. 地图仅突出严重预警车辆。
3. 右侧预警队列切换为严重预警。
4. 底部车辆列表同步筛选存在严重预警的车辆。

### 12.4 从车辆列表批量监管

1. 用户展开底部车辆状态列表。
2. 用户通过企业、申请类型、驾驶模式、在线状态筛选车辆。
3. 地图同步只显示筛选后的车辆。
4. 点击列表行后定位车辆并打开右侧车辆上下文卡。

## 13. 状态与异常

### 13.1 车辆状态

| 状态 | 触发条件 | 展示要求 |
| --- | --- | --- |
| 在线 | 数据正常上报 | 正常状态色 |
| 数据延迟 | 超过配置阈值未上报 | 黄色提示 |
| 离线 | 超过离线阈值未上报 | 灰色车辆点位 |
| 告警中 | 存在未处理预警 | 预警光圈 + 角标 |

### 13.2 预警状态

| 状态 | 展示要求 |
| --- | --- |
| 未处理 | 高亮展示，进入队列顶部 |
| 处理中 | 展示处理中标签 |
| 已处理 | 默认不在待处理队列中展示，可通过筛选查看 |

### 13.3 空状态

| 场景 | 文案 |
| --- | --- |
| 无预警 | 暂无待处理预警 |
| 无筛选结果 | 暂无符合条件的数据 |
| 未选中车辆 | 点击地图车辆或预警卡片，可查看车辆实时状态 |
| 视频不可用 | 视频流不可用，车辆离线或摄像头不可用 |

## 15. 性能与权限

### 15.1 性能要求

1. 车辆位置信息刷新频率不低于 1Hz。
2. 告警和状态变化需准实时推送。
3. 地图需支持缩放、拖拽、聚类展示。
4. 严重预警车辆需保持可见，不被普通聚类淹没。
5. 筛选、图层切换、列表联动响应时间应控制在可感知流畅范围内。

### 15.2 权限要求

1. 仅监管部门和授权人员可访问监控一张图。
2. 视频查看、录像导出、预警处置等操作需根据角色权限控制。
3. 普通查看人员可查看地图、车辆状态和预警摘要。
4. 处置人员可进入预警处置、查看轨迹和视频。

## 16. 验收标准

1. 页面默认以地图为核心，右侧默认展示全局预警队列。
2. 顶部实时态势条正确展示在线车辆、自动驾驶、严重预警、一般预警、当日事故和刷新状态。
3. 车辆 marker 能正确表达驾驶模式、在线状态和预警状态。
4. 严重预警车辆在地图上保持可见，不被普通聚类淹没。
5. 点击预警卡片后，地图自动定位对应车辆，右侧展示选中车辆上下文卡。
6. 点击车辆 marker 后，右侧展示车辆上下文卡和该车预警。
7. 预警队列列表态、展开态、详情态层级清晰，一屏至少可展示 5 条列表态预警。
8. 底部车辆列表支持展开、收起、筛选、分页和地图联动。
9. 图层控制只管理对象显隐，数据筛选按车辆、预警、事故分别组织。
10. 轨迹和视频可从车辆气泡、右侧面板、预警卡片、车辆列表进入。
11. 空状态、离线状态、视频不可用、无筛选结果均有明确提示。
12. 页面在 1366x768 和 1920x1080 分辨率下无明显遮挡、溢出和布局错乱。
