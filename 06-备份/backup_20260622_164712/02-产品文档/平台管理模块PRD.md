## 1. 业务背景与使用角色

### 1.1 业务背景
- 模块定位：平台管理模块是智能网联汽车安全监测平台的**基础支撑模块**，承担平台级元数据标准化管理与用户行为审计职责，为所有业务模块提供统一的数据字典服务，并记录全平台用户的关键操作，保障数据规范性与安全可追溯
- 核心特点：数据字典全局统一定义字段标准与枚举值，避免"同名不同义"；操作日志采用通用操作分类，记录所有用户业务操作，支撑安全审计与责任认定
- 关联模块关系：数据字典被平台所有业务模块（测试示范监管、开放道路资源管理、监测数据分析等）引用；操作日志由各业务模块在关键操作节点调用统一日志服务写入，本模块提供查询与审计入口

### 1.2 使用角色

| 角色 | 该模块中的职责 | 客户端 |
| ---- | ------------ | ------ |
| 系统管理员 | 维护数据字典（新增/编辑/停用/启用/导出），查询与审计操作日志（查看详情/删除/清空日志） | Web管理端 |
| 安全审计员 | 独立查看操作日志，进行合规检查（仅查看权限，不可删除/清空） | Web管理端 |
| 平台所有业务模块 | 在数据交互与操作执行时，向日志服务写入标准化日志 | 服务端 |

### 1.3 核心目标与价值

| 目标维度 | 说明 |
| -------- | ---- |
| 数据标准统一 | 通过数据字典全局管理字段定义与枚举值，确保跨模块数据一致性 |
| 安全审计追溯 | 记录所有用户的业务操作，提供不可篡改的日志查询，支撑安全审计与责任认定 |
| 易于实现 | 操作日志模型设计通用，采用通用操作分类，降低开发落地难度 |
| 生命周期管理 | 字典项支持版本管理，变更历史可追溯，旧版本只读留存 |

## 3. 功能详细设计

#### 3.1.1 页面布局

页面自上而下分为4个区域：

1. **页面标题区**：左侧显示"数据字典管理"标题，右侧放置"导出数据"按钮和"+ 新增字典项"主按钮

4. **弹窗区**：新建/编辑、详情查看、停用确认、启用确认、导出等操作通过模态弹窗完成

#### 3.1.6 交互流程

**新建字典项：**
1. 用户点击"+ 新增字典项"按钮 → 弹出新增弹窗，标题为"新增字典项"，表单为空
2. 填写基本信息与字典值列表
3. 点击"确认提交"：
   - 校验成功 → 保存数据（版本号初始为V1）+ 列表刷新 + Toast提示"字典项创建成功" + 关闭弹窗
   - 校验失败 → 在对应字段下方显示红色错误提示 + 弹窗不关闭
4. 点击"取消" → 关闭弹窗，不保存任何数据

**编辑字典项：**
1. 用户点击操作列的"编辑"按钮 → 弹出编辑弹窗，标题为"编辑字典项"，回填当前所有字段（字典编码置灰不可编辑）
2. 修改字段内容，版本号自动+1
3. 点击"确认提交"：
   - 校验成功 → 保存数据（版本号递增）+ 列表刷新 + Toast提示"字典项更新成功" + 关闭弹窗
   - 校验失败 → 显示错误提示 + 弹窗不关闭
4. 点击"取消" → 关闭弹窗，不保存修改

**查看详情：**
1. 用户点击操作列的"详情"按钮 → 弹出详情弹窗，展示字典项完整信息、字典值明细表、版本记录表
2. 点击"关闭" → 关闭弹窗

**停用操作：**
1. 用户点击"停用"按钮 → 弹出停用确认弹窗
2. 弹窗内容："确定要停用该字典项吗？停用后相关功能将无法使用此字典数据，且所有引用该字典的配置将失效。如需重新启用，请在列表中点击启用按钮。"
3. 点击"确定停用" → 状态变更为"已停用" + 列表刷新 + Toast提示"字典项已停用"
4. 点击"取消" → 关闭弹窗，不执行操作

**启用操作：**
1. 用户点击"启用"按钮 → 弹出启用确认弹窗
2. 弹窗内容："确定要启用该字典项吗？启用后相关功能将可以使用此字典数据。"
3. 点击"确定启用" → 状态变更为"已启用" + 列表刷新 + Toast提示"字典项已启用"
4. 点击"取消" → 关闭弹窗，不执行操作

**导出操作：**
1. 用户点击"导出数据"按钮 → 弹出导出弹窗
2. 选择导出格式（Excel/CSV）和导出范围（当前页数据/全部数据）
3. 点击"开始导出" → 执行导出 + 关闭弹窗 + Toast提示"数据导出成功"
4. 点击"取消" → 关闭弹窗，不执行导出

#### 3.2.4 交互流程

**查询日志：**
1. 用户输入筛选条件（模块名称/操作人员/操作类型/状态/操作日期）
2. 点击"查询"按钮 → 按条件查询 + 列表刷新
3. 点击"重置"按钮 → 清空所有筛选条件 + 列表恢复默认查询

**查看日志详情：**
1. 用户点击操作列的"查看"按钮 → 弹出详情弹窗，展示模块名称、业务类型、请求方法、操作人员、IP地址、请求路径、执行方法、请求参数、返回结果、操作状态、耗时、操作时间
2. 点击"关闭" → 关闭弹窗

**删除操作：**
1. 用户点击"删除"按钮 → 弹出删除确认弹窗
2. 弹窗内容："确认删除该条日志记录吗？删除后将无法恢复，如需保留日志记录请选择导出后再删除。"
3. 点击"确认删除" → 物理删除该条日志 + 列表刷新
4. 点击"取消" → 关闭弹窗，不执行删除

**清空日志操作：**
1. 系统管理员点击"清空日志"危险按钮 → 弹出清空日志弹窗
2. 选择清空日期范围（全部日志 / 指定日期之前的日志）
3. 输入当前登录管理员密码进行安全确认
4. 弹窗内显示红色警告提示："清空日志为不可逆操作，所有操作记录将被物理删除，且在系统审计中将无法追溯。请务必确认数据已导出备份。"
5. 点击"确认清空"：
   - 密码校验成功 → 执行物理清空 + 列表刷新 + Toast提示
   - 密码校验失败 → Toast提示"密码错误，请重新输入" + 弹窗不关闭
6. 点击"取消" → 关闭弹窗，不执行清空

## 4. 业务规则汇总

### 4.1 唯一性规则

| 规则项 | 说明 |
| ------ | ---- |
| 字典编码全局唯一 | 新建字典项时，字典编码不可与已有记录重复；编辑时字典编码只读，不可修改 |
| 字典键名在字典内唯一 | 同一字典项下的字典值列表中，字典键不可重复 |

### 4.2 删除规则

| 规则项 | 说明 |
| ------ | ---- |
| 日志单条删除 | 仅系统管理员可见删除按钮，物理删除，执行前需二次确认，不可恢复 |
| 日志清空 | 仅系统管理员可见清空日志按钮，需输入管理员密码二次确认，执行物理清空，不可恢复 |
| 字典删除 | 本期不开放字典项删除功能，仅支持启用/停用控制 |

### 4.3 状态流转

数据字典状态流转：

```
[新建字典项] → 已启用 ←→ 已停用
                  ↑________↓
                启用 / 停用操作
```

- 新建字典项默认状态为"已启用"
- 已启用状态可执行"停用"操作
- 已停用状态可执行"启用"操作
- 停用后业务模块应忽略该字典项的校验约束，不影响存量数据

### 4.4 字段联动规则

| 触发字段 | 触发条件 | 联动行为 |
| -------- | -------- | -------- |
| "+ 添加字典值"按钮 | 点击 | 在字典值子表格末尾新增一行空白行，排序序号自动递增 |
| 字典值行"删除"按钮 | 点击 | 从子表格中移除当前行 |

### 4.5 计算规则

| 规则项 | 说明 |
| ------ | ---- |
| 版本号递增 | 编辑字典项保存后，版本号自动+1，旧版本数据只读留存 |
| 字典值数量 | 列表中显示的字典值数量为当前字典项包含的字典值条目总数（仅统计已启用数量） |

### 4.6 校验规则

**提交校验（字典项新建/编辑）：**

| 校验项 | 规则 | 提示信息 |
| ------ | ---- | -------- |
| 字典编码 | 不可为空 | "请输入字典编码" |
| 字典编码唯一性 | 新建时不可与已有记录重复 | "字典编码已存在，请重新输入" |
| 字典名称 | 不可为空，最长128字符 | "请输入字典名称" |
| 所属应用 | 不可为空 | "请选择所属应用" |
| 字典键名 | 不可为空 | "请输入字典键名" |
| 字典值类型 | 不可为空 | "请选择字典值类型" |
| 字典值键 | 不可为空 | "字典值第N行字典键不可为空" |
| 字典值值 | 不可为空 | "字典值第N行字典值不可为空" |
| 字典值键唯一 | 同一字典内不可重复 | "字典键'{key}'重复，请修改" |

**清空日志安全校验：**

| 校验项 | 规则 | 提示信息 |
| ------ | ---- | -------- |
| 管理员密码 | 必须输入当前登录管理员密码 | "密码错误，请重新输入" |

### 4.7 权限控制规则

| 操作 | 权限范围说明 |
| ---- | ------------ |
| 数据字典查询/详情/导出 | 系统管理员 |
| 数据字典新建/编辑/停用/启用 | 系统管理员 |
| 操作日志查询/查看详情 | 系统管理员、安全审计员 |
| 操作日志删除 | 系统管理员 |
| 清空日志 | 系统管理员（页面按钮仅该角色可见） |

### 4.8 版本管理规则

| 规则项 | 说明 |
| ------ | ---- |
| 版本号初始值 | 新建字典项时版本号为V1 |
| 版本号递增 | 每次编辑保存后版本号自动+1（V1→V2→V3…） |
| 版本记录保存 | 每次变更记录：版本号、更新时间、操作人、变更内容描述 |
| 旧版本留存 | 旧版本数据只读留存，不可编辑或删除 |

### 4.9 日志写入规则

- 采用AOP统一拦截Controller层请求，结合自定义注解`@Log`标记需要记录的方法
- 注解属性：`module`（模块）、`type`（通用分类）、`description`（描述，支持SpEL表达式）
- 日志通过异步消息队列或线程池写入，不阻塞业务请求
- 耗时计算：拦截器记录开始时间，请求结束后计算差值
- 通用操作分类：查询、导出、新增、修改、删除、审批、其他
