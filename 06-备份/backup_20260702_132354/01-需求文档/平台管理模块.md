## 1. 模块概述

### 1.1 模块定位

平台管理模块是智能网联汽车安全监测平台的**基础支撑模块**，承担平台级元数据标准化管理与用户行为审计职责。本模块为所有业务模块提供统一的数据字典服务，并记录全平台用户的关键操作，保障数据规范性与安全可追溯。

### 1.2 核心价值

- **数据标准统一**：通过数据字典全局管理字段定义与枚举值，避免“同名不同义”。
- **安全审计追溯**：记录所有用户的业务操作，提供不可篡改的日志查询，支撑安全审计与责任认定。
- **易于实现**：操作日志模型设计通用，采用通用操作分类，降低开发落地难度。

### 1.3 适用范围

- **系统管理员**：维护数据字典，查询与审计操作日志。
- **安全审计员**：独立查看操作日志，进行合规检查。
- **平台所有业务模块**：在数据交互与操作执行时，向日志服务写入标准化日志。

## 2. 功能需求

### 2.1 数据字典管理

#### 2.1.1 功能概述

统一管理平台全量业务元数据，集中定义各模块使用的字段标准，支持字典项的生命周期管理。字典变更后，相关业务模块需实时感知。

#### 2.1.2 字典项数据结构

| 字段名     | 类型           |  必填 | 说明                        | 示例                                                      |
| :------ | :----------- | :-: | :------------------------ | :------------------------------------------------------ |
| 字典编码    | varchar(64)  |  是  | 全局唯一标识，如`VEHICLE_TYPE`    | VEHICLE\_TYPE                                           |
| 字典名称    | varchar(128) |  是  | 如“车辆类型”                   | 车辆类型                                                    |
| 所属模块    | varchar(64)  |  是  | 如“测试示范监管”                 | 测试示范监管                                                  |
| 字段名     | varchar(64)  |  是  | 对应数据库字段                   | vehicle\_type                                           |
| 字段类型    | varchar(32)  |  是  | 字符串、整数、小数、日期、布尔           | 字符串                                                     |
| 字段长度    | int          |  否  | <br />                    | 50                                                      |
| 是否必填    | boolean      |  是  | <br />                    | 是                                                       |
| 默认值     | varchar(128) |  否  | <br />                    | 乘用车                                                     |
| 字典值集合   | json/list    |  是  | 枚举值列表，每项含：值编码、值名称、是否启用、排序 | \[{"code":"CAR","name":"乘用车","enabled":true,"order":1}] |
| 状态      | varchar(16)  |  是  | 启用 / 停用                   | 启用                                                      |
| 版本      | int          |  是  | 修改后自动+1                   | 3                                                       |
| 创建/更新时间 | datetime     |  是  | <br />                    | <br />                                                  |
| 创建/更新人  | varchar(64)  |  是  | <br />                    | <br />                                                  |

#### 2.1.3 功能点列表

| 序号 | 功能     | 描述                                  |
| :- | :----- | :---------------------------------- |
| 1  | 字典列表查询 | 支持按字典编码、名称、模块、状态筛选，分页展示，默认按更新时间倒序   |
| 2  | 新建字典项  | 填写所有必填字段，字典编码唯一性校验，字典值支持表格编辑或JSON导入 |
| 3  | 编辑字典项  | 支持修改除编码外的字段，版本号自动递增，旧版本只读留存         |
| 4  | 启用/停用  | 停用后业务模块应忽略该校验，不影响存量数据               |
| 5  | 历史版本查看 | 展示修改时间、修改人、变更前后Diff对比               |
| 6  | 导出     | 支持导出为Excel/CSV                      |

### 2.2 操作日志管理

#### 2.2.1 功能概述

全平台统一的用户操作日志查询与审计中心。采用**通用的操作分类方式**，记录用户通过Web端发起的业务流程操作（查询、导出、新增、修改、删除等），由各业务模块在关键操作节点调用统一日志服务写入。

#### 2.2.2 页面布局

页面分为**搜索筛选区域**和**日志列表区域**两部分，参考截图样式设计。

#### 2.2.3 搜索筛选栏

位于页面顶部，包含以下筛选条件与操作按钮：

| 筛选项/按钮  | 组件类型 | 说明                                  |
| :------ | :--- | :---------------------------------- |
| 模块名称    | 输入框  | 支持模糊搜索，placeholder：“请输入模块名称”        |
| 操作人员    | 输入框  | 支持模糊搜索，placeholder：“请输入操作人员”        |
| 状态      | 下拉选择 | 选项：全部 / 正常 / 异常，placeholder：“请选择状态” |
| \[Q搜索]  | 按钮   | 触发查询                                |
| \[重置]   | 按钮   | 清空所有筛选条件并恢复默认                       |
| \[清空日志] | 按钮   | 点击后弹出二次确认框，确认后物理删除全部日志（仅超级管理员可见）    |

#### 2.2.4 日志列表展示

表格列定义（严格按照截图字段）：

| 列名   | 字段              | 说明                         |
| :--- | :-------------- | :------------------------- |
| ID   | log\_id         | 自增或分布式ID，数字类型              |
| 模块名称 | module\_name    | 如“用户管理”、“测试示范监管”           |
| 业务类型 | business\_type  | 通用分类：查询、导出、新增、修改、删除、审批、其他  |
| 请求方式 | request\_method | HTTP方法：GET、POST、PUT、DELETE |
| 操作人员 | operator        | 登录账号                       |
| IP地址 | ip\_address     | 操作来源IP                     |
| 状态   | status          | 正常 / 异常，正常用绿色标记，异常用红色标记    |
| 耗时   | duration        | 单位ms，如“139ms”              |
| 操作时间 | create\_time    | 格式 YYYY-MM-DD HH:mm:ss     |
| 操作   | 按钮              | \[详情] \[删除]                |

- **排序**：默认按操作时间倒序
- **删除按钮**：仅系统管理员可见，删除为物理删除，需二次确认
- **状态列**：正常显示绿色圆点+“正常”，异常显示红色圆点+“异常”

#### 2.2.5 分页器

位于列表底部，样式与截图一致：

- 左侧显示“共xx条”
- 中间为页码按钮，支持前后翻页
- 右侧包含页容量选择器（10/page，可切换20/50/100）
- 最右侧为GoTo跳页输入框，输入页码后回车跳转

#### 2.2.7 操作分类定义（通用）

为简化实现，采用以下通用分类，由业务模块调用日志服务时动态传入：

| 通用分类 | 描述模板       | 示例          |
| :--- | :--------- | :---------- |
| 查询   | 查询{模块}数据   | 查询车辆列表      |
| 导出   | 导出{模块}数据   | 导出事故报告      |
| 新增   | 新增{模块}{对象} | 新增电子围栏      |
| 修改   | 修改{模块}{对象} | 修改道路资源信息    |
| 删除   | 删除{模块}{对象} | 删除字典项       |
| 审批   | {模块}-{动作}  | 测试示范监管-审批通过 |
| 其他   | 自定义        | 用户登录、清空日志等  |

#### 2.2.8 日志写入机制

- 采用AOP统一拦截Controller层请求，结合自定义注解`@Log`标记需要记录的方法。
- 注解属性：`module`（模块）、`type`（通用分类）、`description`（描述，支持SpEL表达式）。
- 日志通过异步消息队列或线程池写入，不阻塞业务请求。
- 耗时计算：拦截器记录开始时间，请求结束后计算差值。

## 3. 通用交互规范

- 所有列表页均提供搜索、分页、导出（本模块日志导出字段与列表一致）。
- 危险操作（清空日志、删除日志）需二次确认。
- 详情使用模态框展示。
- 空状态显示“暂无数据”提示，加载状态显示Spin动画。

***

## 4. 非功能需求

- 日志写入不阻塞业务，写入延迟<50ms。
- 单表日志量可能较大，需按时间分表或使用Elasticsearch，查询性能需达标。
- 日志保留周期≥6个月，支持归档。
- 数据字典更新后，各节点缓存刷新延迟≤10秒。
