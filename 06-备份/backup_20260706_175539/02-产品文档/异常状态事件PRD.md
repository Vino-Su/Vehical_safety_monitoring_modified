## 名词解释

| 名词/缩写  | 解释                                                                             |
| ------ | ------------------------------------------------------------------------------ |
| 预警     | 系统通过实时计算任务检测到车辆运行过程中出现安全/技术风险时自动生成的早期提示，属于风险预判类事件（共 3 类：车辆运行异常、碰撞事件、车载终端异常）    |
| 告警     | 系统通过实时计算任务检测到车辆运行过程中发生合规性违规行为时自动生成的违规通知，属于合规违反类事件（共 4 类：临牌有效期、围栏越界、时段合规、安全员到岗） |
| 触发条件   | 系统判定生成预警/告警的多维度联合判定规则，包含数据生成时间、车速、驾驶自动化状态、定位、档位、安全员状态等字段的组合判断                  |
| 电子围栏   | 在地图上划定的虚拟地理边界，用于限制或监控车辆行驶范围                                                    |
| 自动驾驶模式 | 车辆驾驶自动化状态为自动驾驶（非人工驾驶），为多数事件触发的前置条件                                             |
| 数据生成时间 | 车辆实时数据包中携带的时间戳，表示该数据包的实际采集时刻                                                   |
| VIN 码  | 车辆识别代号（Vehicle Identification Number），17 位字符唯一标识                               |

## 1. 业务背景与使用角色

### 1.1 业务背景
- **模块定位**：异常状态事件是测试示范监管模块（F15）的核心子功能，承担智能网联汽车运行过程中 7 类风险事件的实时检测、展示与处置管理
- **核心特点**：基于车辆实时数据包与多模型识别引擎，通过预设触发条件自动生成事件；7 类事件按性质分为**预警**（安全/技术风险预判）和**告警**（合规性违规通知）两大类
- **事件分类体系**：
  - **预警（3 类）**：车辆运行异常预警、车辆碰撞事件预警、车载终端异常预警 —— 侧重于安全风险的早期发现与预判
  - **告警（4 类）**：临牌有效期告警、电子围栏越界告警、测试时段合规性告警、安全员到岗状态告警 —— 侧重于合规性违规行为的检测与通知
- **关联模块关系**：
  - 监控一张图（F13）：一张图中以醒目图标标注存在预警/告警的车辆，支持点击跳转至本页面
  - 车辆运行监测（F14）：故障信息采集后推送至本页面生成终端异常预警
  - 电子围栏管理（F19）：围栏地理数据用于电子围栏越界告警的定位比对
  - 监测数据分析模块：事件数据汇聚至告警统计分析（F02），用于趋势分析与企业排名
  - 车辆信息登记（F02）：临牌有效期数据来源，录入临时牌照时关联有效期
- **数据来源**：车辆实时数据包（数据生成时间、车速、驾驶自动化状态、经纬度、档位、安全员状态等）、车载总线故障数据、碰撞事件识别模型输出、电子围栏地理数据、安全员排班数据

### 1.2 使用角色

| 角色 | 该模块中的职责 | 操作权限 | 客户端 |
| ---- | ------------ | -------- | ------ |
| 政府监管部门（经信/交通/公安） | 查看全部预警/告警、处理事件、监督处置情况 | 查看 + 处理 | Web 端 |
| 第三方专业管理机构 | 查看全部预警/告警、处理事件、跟踪处置进度 | 查看 + 处理 | Web 端 |
| 产业运营主体（企业端） | 查看本企业车辆的预警/告警信息 | 仅查看 | Web 端 |

### 1.3 核心目标与价值

| 目标维度 | 说明 |
| -------- | ---- |
| 实时感知 | 对车辆运行过程中的 7 类风险事件（3 类预警 + 4 类告警）进行实时检测与自动推送，从触发到前端展示 ≤ 3 秒 |
| 闭环处置 | 提供"待处理→处理中→已处理"的完整处置流程，确保每条事件可跟踪、可追溯 |
| 合规监管 | 通过告警类事件（临牌有效期、测试时段、电子围栏、安全员到岗），保障测试活动依法依规开展 |
| 风险防控 | 通过预警类事件（车辆运行异常、碰撞事件、车载终端异常），实现安全风险早发现、早处置 |

## 3. 功能详细设计

### 3.1 F15-1 异常状态事件页面

#### 3.1.1 页面布局

页面自上而下分为 4 个区域：
1. **页面标题区**：标题"异常状态事件"
2. **统计卡片区**（4 列网格）：未处理事件数、正在处理事件数、已处理事件数、累计事件总数
3. **Tab 页签区**（7 个，按分类分组排列）：
   - **预警组（3 个 Tab）**：车辆运行异常预警、车辆碰撞事件预警、车载终端异常预警
   - **告警组（4 个 Tab）**：临牌有效期告警、电子围栏越界告警、测试时段合规性告警、安全员到岗状态告警
4. **内容区**：每个 Tab 下包含 查询筛选区 + 数据列表区，Tab 切换时内容区联动刷新

**统计卡片配置：**

| 序号 | 卡片名称 | 图标色值 | 统计口径 |
| ---- | -------- | -------- | -------- |
| 1 | 未处理事件数 | 红色(#ff4d4f) | 当日处理状态为"待处理"的全部事件数量（预警 + 告警） |
| 2 | 正在处理事件数 | 蓝色(#1890ff) | 当日处理状态为"处理中"的全部事件数量 |
| 3 | 已处理事件数 | 绿色(#52c41a) | 当日处理状态为"已处理"的全部事件数量 |
| 4 | 累计事件总数 | 灰色(#00000073) | 历史全部事件数量 |

**Tab 分组与排列规则：**
- 预警组 Tab 标签可增加蓝色圆点标识，告警组 Tab 标签可增加橙色圆点标识，便于视觉区分
- 默认选中第一个 Tab（车辆运行异常预警）
- 点击 Tab 标签切换内容区，切换后自动以该 Tab 默认筛选条件重新加载列表数据

#### 3.1.2 查询筛选区

各 Tab 共用基础筛选字段，部分 Tab 有差异化筛选字段。以下为基础字段，各 Tab 差异化字段在对应 Tab 章节中补充。

**基础筛选字段（所有 Tab 共用）：**

| 元素 | 类型 | 配置说明 |
| ---- | ---- | -------- |
| 车牌号 | 文本输入 | 精确匹配，placeholder"请输入车牌号" |
| VIN 码 | 文本输入 | 精确匹配，placeholder"请输入VIN码" |
| 处理状态 | 下拉单选 | 枚举值：全部、待处理、处理中、已处理 |
| 发生日期 | 日期范围选择器 | 支持起止日期选择 |

**操作按钮：**

| 按钮 | 触发行为 | 可见角色 |
| ---- | -------- | -------- |
| 查询 | 按当前筛选条件重新加载列表 | 全部 |
| 重置 | 清空所有筛选条件，恢复默认值后重新加载 | 全部 |
| 导出 | 导出当前筛选条件下的事件数据（见附录 A.3） | 政府监管部门、第三方专业管理机构 |

#### 3.1.3 数据列表区

**列表显示字段（所有 Tab 共用）：**

| 序号  | 字段名    | 字段标识        | 宽度建议  | 说明                                 |
| --- | ------ | ----------- | ----- | ---------------------------------- |
| 1   | 事件编号   | id          | 200px | 格式见 4.2 编号生成规则（预警=EW 前缀，告警=ALT 前缀） |

**排序规则：** 默认按发生时间倒序。

**操作列按钮：**

| 操作 | 触发行为 | 可见角色 |
| ---- | -------- | -------- |
| 查看详情 | 打开事件详情弹窗，展示该条事件的完整信息 | 全部 |

**行交互规则：** hover 高亮；待处理状态行以浅红色背景(#fff2f0)标识，便于视觉区分（见附录 A.5 复选框组件——本期不启用行复选框，无需批量操作）。

## 4. 业务规则汇总

### 4.1 事件触发规则

**预警类（3 类）—— 安全/技术风险预判：**

| 事件类型 | 事件分类 | 触发条件（需同时满足） |
| -------- | -------- | ---------------------- |
| 车辆运行异常预警 | 预警 | 车速/加速度超限阈值 **或** 驶出规定路段 **或** 偏离指定路线 |
| 车辆碰撞事件预警 | 预警 | 碰撞识别模型判定发生碰撞/冲击事件 |
| 车载终端异常预警 | 预警 | 终端数据中断/停滞/异常传输 **或** 车载总线报告硬件故障 |

**告警类（4 类）—— 合规性违规通知：**

| 事件类型      | 事件分类 | 触发条件（需同时满足）                                                            |
| --------- | ---- | ---------------------------------------------------------------------- |
| 临牌有效期告警   | 告警   | `数据生成时间 ∉ 有效期` **且** `驾驶模式 = 自动驾驶` **且** `车速 ≠ 0`                      |
| 电子围栏越界告警  | 告警   | `驾驶模式 = 自动驾驶` **且** `定位 ∉ 围栏范围` **且** `档位 ∈ {D,S,R}` **且** `车速 ≠ 0`    |
| 测试时段合规性告警 | 告警   | `数据生成时间 ∉ 测试时段` **且** `驾驶模式 = 自动驾驶` **且** `车速 ≠ 0`                     |
| 安全员到岗状态告警 | 告警   | `数据生成时间 ∈ 排班时段` **且** `驾驶模式 = 自动驾驶` **且** `车速 ≠ 0` **且** `安全员状态 = 未到岗` |

### 4.2 编号生成规则

| 规则项 | 说明 |
| ------ | ---- |
| 通用格式 | {前缀}{类型编号}-{YYYYMMDD}-{当日序号} |
| 预警前缀 | EW（Early Warning） |
| 告警前缀 | ALT（ALert） |
| 预警类型编号 | EW01=车辆运行异常、EW02=碰撞事件、EW03=终端异常 |
| 告警类型编号 | ALT04=临牌有效期、ALT05=围栏越界、ALT06=时段合规、ALT07=安全员到岗 |
| 日期部分 | 事件发生的日期，YYYYMMDD 格式 |
| 序号部分 | 当日同类事件的流水号，从 001 递增，3 位补零 |
| 预警示例 | EW01-20260511-001（2026年5月11日第1条车辆运行异常预警） |
| 告警示例 | ALT04-20260511-001（2026年5月11日第1条临牌有效期告警） |

### 4.3 处理状态流转

```
待处理 ──→ 处理中 ──→ 已处理
```

| 状态 | 标签颜色 | 可流转至 | 操作角色 |
| ---- | -------- | -------- | -------- |
| 待处理 | 橙色(#faad14) | 处理中 | 政府监管部门、第三方专业管理机构 |
| 处理中 | 蓝色(#1890ff) | 已处理 | 政府监管部门、第三方专业管理机构 |
| 已处理 | 绿色(#52c41a) | —（终态） | — |

**状态流转规则：**
- 状态只能单向流转（待处理→处理中→已处理），不可逆向
- 状态变更时需记录处理人、处理时间、处理备注
- 企业端角色仅可查看状态，不可变更

### 4.4 统计卡片刷新规则

- 统计数据与当前 Tab 无关，为全部 7 类事件（预警 + 告警）的汇总统计
- "今日"的定义：自然日（00:00:00 ~ 23:59:59）
- 数据刷新频率：准实时，页面打开时自动加载，切换 Tab 时刷新列表但不刷新统计卡片

### 4.5 权限规则

| 操作 | 政府监管部门 | 第三方专业管理机构 | 产业运营主体 |
| ---- | ------------ | ------------------ | ------------ |
| 查看全部事件 | ✓ | ✓ | ✗（仅本企业车辆） |
| 查看事件详情 | ✓ | ✓ | ✓（仅本企业车辆） |
| 变更处理状态 | ✓ | ✓ | ✗ |
| 导出事件数据 | ✓ | ✓ | ✗ |
