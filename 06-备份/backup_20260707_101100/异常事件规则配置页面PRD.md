# 名词解释

| 名词/缩写 | 解释 |
| ---------- | ---- |
| 系统预置规则 | 平台内置的异常事件判定规则模板，用户不可新增、删除或修改核心触发逻辑，仅可配置启停、阈值、适用范围、豁免条件等策略参数 |
| 规则配置实例 | 基于某条系统预置规则形成的当前生效配置，包含启停状态、参数值、适用对象、适用范围、生效时段、通知策略等 |
| 规则子类型 | 一级事件类型下的具体规则项，如超时停靠、车速超限、允许通行区域越界、禁行区域闯入、高峰时段运行等 |
| 阈值参数 | 规则触发时用于判定的可配置数值，如持续时长、速度容差、位移阈值、越界缓冲距离、连续采样次数、模型置信度等 |
| 豁免条件 | 满足后不触发异常事件的排除条件，如人工接管、任务结束、计划停靠、停车场、充电站、维修区、装卸点等 |
| 去重周期 | 同一车辆、同一规则、同一未恢复事件在指定时间内不重复生成事件的控制周期 |
| 恢复条件 | 异常事件从触发状态恢复为正常状态的条件，如车辆回到围栏内、车速恢复、终端恢复上报、安全员重新在线等 |
| 规则版本 | 每次保存规则配置后生成的版本快照，用于记录配置变更并支持事件触发追溯 |

## 1 业务背景与使用角色
### 1.1 业务背景
- 模块定位：异常事件规则配置页面是测试示范监管模块的规则策略配置中心，用于对平台系统预置的预警/告警规则进行启停、阈值、适用范围、生效时段、豁免条件、去重恢复和通知策略配置。
- 核心特点：规则模板由系统内置并保持监管口径稳定，用户不支持新建、删除或修改核心判定逻辑，仅对规则策略参数进行配置。
- 关联模块关系：异常状态事件页面引用规则配置生成事件；监控一张图引用规则触发结果进行地图高亮和弹窗展示；电子围栏管理提供围栏范围、围栏类型和围栏状态；道路资源管理提供道路限速、开放状态和授权路段；车辆/企业/人员信息登记提供适用对象和安全员状态；监测数据分析模块汇总规则触发结果。
- 建设必要性：现有异常事件体系已细化到超时停靠、车速超限、电子围栏子类型、高峰时段、安全员离岗等规则项，不同车辆类型、道路、区域和场景需要差异化阈值，需提供统一配置入口降低误报并提升规则可追溯性。
### 1.2 使用角色

| 角色 | 该模块中的职责 | 客户端 |
| ---- | ------------ | ------ |
| 政府监管部门 | 查看全部规则配置，启用/停用规则，调整关键参数，查看版本历史 | Web 端 |
| 第三方专业管理机构 | 查看全部规则配置，按授权范围协助配置规则参数，查看版本历史 | Web 端 |
| 平台运维管理员 | 维护系统预置规则模板、处理配置异常、协助恢复默认配置 | Web 端 |
| 产业运营主体 | 不进入规则配置页面，仅在异常状态事件和监控一张图中查看本企业触发结果 | Web 端 |

### 1.3 核心目标与价值

| 目标维度 | 说明 |
| -------- | ---- |
| 监管口径稳定 | 通过系统预置规则模板固定一级事件类型和核心判定逻辑，避免用户自由创建规则导致口径失控 |
| 策略灵活配置 | 支持按车辆类型、企业、道路、围栏、场景和时段配置阈值参数，实现差异化监管 |
| 降低误报 | 通过豁免条件、连续采样、缓冲距离、去重周期和恢复条件减少边界漂移、短时波动和重复触发 |
| 可追溯 | 每次配置变更生成版本记录，事件触发时记录规则编号和配置版本，便于复盘和责任追踪 |
| 可运营 | 支持规则状态、参数摘要和版本历史展示，帮助监管人员识别需优化配置的规则 |

## 3 功能详细设计
### 3.1 ARC-1 规则配置列表
#### 3.1.1 页面布局

页面自上而下分为 3 个区域：

1. **页面标题区**：仅展示标题“异常事件规则配置”，标题下不展示说明文字。
2. **查询筛选区**：默认展示关键词、事件分类、事件类型、规则状态 4 个条件，其余条件通过“展开/收起”控制。
3. **规则列表区**：表格展示所有系统预置规则，分页区见附录 A.1；列表上方不展示列表标题、说明文字或统计摘要。
#### 3.1.2 查询筛选区

| 元素     | 类型   | 配置说明                                                                   |
| ------ | ---- | ---------------------------------------------------------------------- |
| 关键词    | 文本输入 | 支持规则编号、事件类型、规则子类型模糊搜索                                                        |
| 事件分类   | 下拉单选 | 枚举值：全部、预警、告警                                                           |
| 事件类型   | 下拉单选 | 枚举值：全部、车辆运行异常预警、车辆碰撞事件预警、车载终端异常预警、临牌有效期告警、电子围栏越界告警、测试时段合规性告警、安全员到岗状态告警 |
| 规则状态   | 下拉单选 | 枚举值：全部、启用、停用                                                           |
| 适用车辆类型 | 下拉单选 | 枚举值：全部、乘用车、商用车、专用作业车、功能型低速无人车、其他                                       |
| 适用范围   | 下拉单选 | 枚举值：全部、道路、电子围栏、企业、车辆、全局                                                |
| 是否关键规则 | 下拉单选 | 枚举值：全部、是、否                                                            |
| 当前版本   | 文本输入 | 支持按规则当前配置版本精确查询，如 V1.3                                               |

**查询条件显示规则：**

- 默认展示关键词、事件分类、事件类型、规则状态 4 项。
- 适用车辆类型、适用范围、是否关键规则、当前版本默认收起，通过“展开/收起”按钮控制显示。
**操作按钮：**

| 按钮   | 触发行为                         | 可见角色                     |
| ---- | ---------------------------- | ------------------------ |
| 查询   | 按当前筛选条件刷新规则列表                | 政府监管部门、第三方专业管理机构、平台运维管理员 |
| 重置   | 清空筛选条件并恢复默认列表                | 政府监管部门、第三方专业管理机构、平台运维管理员 |
| 展开/收起 | 展开或收起默认隐藏的筛选条件，不刷新列表 | 政府监管部门、第三方专业管理机构、平台运维管理员 |

| 配置    | 打开规则配置弹窗，允许编辑可配置字段，支持全屏展示      | 政府监管部门、第三方专业管理机构（授权范围内）、平台运维管理员 |
| 启用/停用 | 切换规则状态，关键规则停用需二次确认并填写原因 | 政府监管部门、平台运维管理员                  |
| 版本历史  | 打开版本历史弹窗                | 政府监管部门、第三方专业管理机构、平台运维管理员        |

**行交互规则：** 行 hover 高亮；停用规则整行文字弱化

#### 3.1.4 前置校验

| 校验条件 | 提示信息 | 阻断操作 |
| -------- | -------- | -------- |
| 无权限角色点击启用/停用 | 当前角色无权执行启用/停用操作 | 阻断操作 |
| 停用原因超过长度限制 | 停用原因最多 200 字符 | 阻断保存 |

#### 3.1.5 交互流程
1. 用户进入页面 → 系统加载规则列表 → 默认展示全部系统预置规则。
2. 用户输入筛选条件并点击“查询” → 系统刷新规则列表 → Toast 提示“查询完成”。
3. 用户点击“展开/收起” → 系统展开或收起默认隐藏的筛选条件，不刷新列表。
4. 用户点击“配置” → 打开规则配置弹窗 → 系统加载规则模板和当前配置版本。
5. 用户点击“启用/停用”：
   - 弹出二次确认，支持填写变更原因 → 确认后保存状态并刷新列表。
6. 用户点击“恢复默认参数” → 弹出二次确认 → 确认后恢复系统默认配置并生成新版本。

### 3.2 ARC-2 规则配置弹窗
#### 3.2.1 页面布局

弹窗自上而下分为 7 个区域，弹窗支持普通模式和全屏模式：

1. **规则只读信息区**：展示规则编号、事件分类、事件类型、规则子类型、核心触发说明。
2. **启停设置区**：配置规则是否启用。
3. **适用范围区**：配置适用企业、车辆类型、车辆、道路、围栏和业务场景。
4. **参数配置区**：根据规则子类型动态展示可配置参数。
5. **生效时段区**：配置全天、指定时间段、工作日、节假日或高峰时段。
6. **豁免与去重恢复区**：配置豁免条件、去重周期、恢复条件。
7. **通知与保存区**：配置通知策略并提交保存。

#### 3.2.4 交互流程
1. 用户点击规则列表“配置” → 打开规则配置弹窗 → 系统展示规则只读信息和当前配置。
2. 用户调整启停状态、适用范围、阈值参数、生效时段、豁免条件、去重恢复和通知策略。
3. 用户点击“恢复默认参数” → 弹出二次确认 → 确认后将当前表单恢复为系统默认值，未点击保存前不落库。
4. 用户点击“保存配置”：
   - 校验成功 → 保存配置实例 + 生成新版本号 + 记录变更日志 + 刷新规则列表 + Toast 提示“规则配置已保存”。
   - 校验失败 → 在对应字段下方显示错误提示 + 弹窗不关闭。
5. 用户点击“取消” → 如表单有未保存变更，弹出离开确认；确认离开后不保存修改并关闭弹窗。
